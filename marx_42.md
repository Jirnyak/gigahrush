## Текущая задача
**ИИ Багфикс: Духи бьются об столы. Убрать коллизии для летунов с низкими объектами.**

### Контекст задачи
Духи (Spirit, `MonsterKind.SPIRIT`) и другие летающие/эфирные мобы (например, ложный дух) технически должны проходить сквозь низкие или проницаемые препятствия. В текущей реализации они сталкиваются со столами, бочками и другими `pathBlockers`, хотя могут проходить сквозь стены, если у них есть флаг `noclip`.
Задача состоит в том, чтобы заставить мобов с флагами `'flying'` или `'noclip'` полностью игнорировать массив препятствий `world.pathBlockers` при расчете перемещения.

### Конкретные файлы и паттерны
- **`src/entities/spirit.ts`**: Проверить `MonsterDef`. Убедиться, что в `aiFlags` присутствует `'flying'` или `'noclip'`.
- **`src/systems/ai/monster.ts`**: В цикле движения (movement loop), если у монстра есть эти флаги, не делать проверок с `World.pathBlockers`.
- **`src/core/world.ts`**: `pathBlockers` хранится как битовая маска или 8-byte row per cell. Убедитесь, что логика движения летающих юнитов просто пропускает чтение этого массива.
- **Прочие сущности**: Убедитесь, что `LOZHNYY_DUKH` также имеет корректные флаги.

### Детальная спецификация по Архитектуре и Реализации (Строго обязательно к исполнению)

#### Шаг 1: Добавление флагов к летающим монстрам
Перейдите в файлы определений (например, `src/entities/spirit.ts`) и убедитесь, что флаги прописаны корректно.
```ts
// src/entities/spirit.ts
export const DEF: MonsterDef = {
  kind: MonsterKind.SPIRIT,
  // ...
  aiFlags: ['melee', 'flying', 'noclip'], // Обязательно наличие noclip или flying
  // ...
};
```
Сделайте то же самое для ложного духа (`src/entities/lozhnyy_dukh.ts` или аналогичный файл).

#### Шаг 2: Изменение системы движения и коллизий
В функции, обрабатывающей перемещение (например, `resolveMovement` в `src/systems/physics.ts` или `src/systems/ai/monster.ts`), добавьте проверку на флаги.
```ts
// Пример модификации в системе движения
function canMoveTo(entity: Entity, targetX: number, targetY: number, world: World): boolean {
  const isFlying = hasAiFlag(entity, 'flying');
  const isNoclip = hasAiFlag(entity, 'noclip');

  if (isNoclip) {
    return true; // Полностью игнорирует всё
  }

  // Для обычных летающих - игнорируем pathBlockers (низкие объекты), но не стены
  const cellType = world.getCell(Math.floor(targetX), Math.floor(targetY));
  if (isSolidWall(cellType)) {
    return false;
  }

  if (isFlying) {
    // Игнорируем проверку world.pathBlockers
    return true;
  }

  // Для наземных сущностей - полная проверка
  if (isPathBlockedByFeature(world, targetX, targetY)) {
    return false;
  }

  return true;
}
```

#### Шаг 3: Проверка Raycast-проверок LOS (Line of Sight)
Если духи могут стрелять/атаковать через препятствия, убедитесь, что их `hasLineOfSight` функция также игнорирует препятствия на уровне столов. (Для духов, скорее всего, это уже так).

#### Шаг 4: Кэш Nav Tree
Для летающих юнитов, возможно, потребуется игнорировать штрафы от `pathBlockers` в pathfinding, но поскольку они летят по прямой к игроку (летуны обычно используют прямой steer-to-target или игнорируют навмеш), убедитесь, что они не пытаются обходить стол по навмешу.
```ts
// src/systems/ai/steering.ts
if (hasAiFlag(entity, 'flying')) {
   // Steer directly to target, ignore nav mesh node traversal if blocked only by features
   return steerDirectly(entity, target, world);
}
```
