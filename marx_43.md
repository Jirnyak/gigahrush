## Текущая задача
**Боевка: Пси Культистов. Убедиться, что они кастуют заявленные пси-атаки.**

### Контекст задачи
Фракция Культисты (`Faction.CULTIST`) по лору использует пси-способности (пси-удар, пси-щит). Однако сейчас они либо не используют их в бою, либо им вообще не выдается PSI-экипировка (PSI clot) при спавне.
Вам необходимо починить генерацию лута/экипировки для Культистов, чтобы они получали PSI-способности (`psi_strike`, `psi_storm`), а также убедиться, что их боевой ИИ (`ai/npc_utility.ts` или `ai/monster.ts`) корректно применяет эти способности в бою.

### Конкретные файлы и паттерны
- **`src/systems/procedural_loot.ts`** / **`src/data/alife_generation.ts`**: Проверить и исправить генерацию loadout для `Faction.CULTIST`. Выдавать PSI-узел в `tool` или `weapon` слот.
- **`src/data/psi.ts`**: Проверить характеристики PSI-оружия.
- **`src/systems/ai/npc_utility.ts`**: Добавить/исправить intent для использования PSI-оружия в бою, если оно есть в инвентаре.

### Детальная спецификация по Архитектуре и Реализации (Строго обязательно к исполнению)

#### Шаг 1: Фикс выдачи экипировки (Loadout Generation)
Перейдите в файл, где генерируются лоадауты для фракций (скорее всего `src/data/alife_generation.ts` или аналогичный).
Для фракции Культистов с определенной вероятностью или обязательно добавляйте PSI-способность.
```ts
// src/data/alife_generation.ts
export function generateFactionLoadout(faction: Faction, rank: number): Loadout {
  const loadout: Loadout = { weapons: [], ammo: {}, tools: [] };
  
  // ...
  if (faction === Faction.CULTIST) {
    // Культисты рангом выше 1 имеют 50% шанс получить psi_strike
    if (rank > 1 && Math.random() > 0.5) {
      loadout.tools.push('psi_strike');
    } else {
      loadout.tools.push('psi_shield'); // Базовый инструмент
    }
  }
  // ...
  return loadout;
}
```

#### Шаг 2: Использование PSI в боевом ИИ
NPC в бою используют utility AI (микро-цели).
В `src/systems/ai/npc_utility.ts` (или аналогичном файле с ИИ) добавьте проверку: если NPC культист, и у него в инвентаре есть PSI-tool, он должен его использовать.
```ts
// src/systems/ai/npc_utility.ts (combat loop)
function selectBestCombatAction(npc: Npc, target: Entity): CombatAction {
  // Проверяем наличие PSI
  const psiTool = npc.inventory.tools.find(t => isPsiWeapon(t));
  
  if (psiTool && isPsiCooldownReady(npc, psiTool)) {
    const dist = calculateDist2(npc.x, npc.y, target.x, target.y);
    const psiDef = getPsiDef(psiTool);
    
    // Если в радиусе поражения - кастуем
    if (dist <= psiDef.range * psiDef.range) {
       return { type: 'use_psi', toolId: psiTool, target: target.id };
    }
  }
  
  // Иначе стреляем из обычного оружия (существующая логика)
  return getStandardWeaponAction(npc, target);
}
```

#### Шаг 3: Проверка характеристик в `src/data/psi.ts`
Убедитесь, что `'psi_strike'` и `'psi_shield'` существуют в определении PSI и имеют параметры `range`, `cooldown`, `damage` и т.д.
Если нет - добавьте их в реестр.
```ts
// src/data/psi.ts
export const PSI_ABILITIES = {
  'psi_strike': { name: 'Пси-удар', range: 5, damage: 15, cooldown: 5000, type: 'attack' },
  'psi_shield': { name: 'Пси-щит', duration: 10000, armorBonus: 5, cooldown: 15000, type: 'buff' }
};
```

#### Шаг 4: Прокидывание события атаки
Когда ИИ решает применить пси, убедитесь, что в систему эвентов прокидывается `publishEvent('psi_cast', { actor: npc.id, skill: 'psi_strike' })`, чтобы рендерер мог отрисовать партиклы.
