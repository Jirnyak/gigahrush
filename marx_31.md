## Текущая задача
**Обрушенный Сектор: Генератор органической геометрии (design_floors/collapsed_sector.ts) через клеточные автоматы.**



### Контекст задачи
Обрушенный Сектор — этаж с органической геометрией, возникшей после катастрофического самосбора. Несколько уровней хруща схлопнулись, бетон оплыл, арматура обнажилась. Это НЕ пещера — это последствия деструкции бетонной мегаструктуры. Cellular automata создают именно такой эффект: неровные стены, причудливые проходы, тупики из обломков.

### Конкретные файлы и паттерны
- **`src/gen/design_floors/collapsed_sector.ts`** [НОВЫЙ ФАЙЛ]: Генерация обрушенного сектора через cellular automata:
  1. Заполнить 1024x1024 случайно (45% стен).
  2. 4-5 итераций правила B5678/S45678 (сглаживание).
  3. Оставить только largest connected component (flood fill).
  4. Добавить обнажённые коммуникации: кластеры стен с тегом `'exposed_pipes'` (трубы/кабели — ресурсы при разрушении).
- **Текстуры**: `Tex.CONCRETE` и `Tex.ROTTEN` (оплывший бетон), `Tex.F_CONCRETE` (щебень). Используйте существующие текстуры — это бетон, не камень.
- **`src/data/design_floors.ts`**: Выберите свободный `z`.
- **Seed**: Используйте `SeedRng` / `xorshift32` из `src/core/rand.ts`. НЕ `Math.random()`.

### Детальный Пошаговый План Реализации (Workflow)

#### Шаг 1: Настройка клеточных автоматов (src/gen/cellular_automata.ts)
- Создайте универсальную функцию `runCellularAutomata`, принимающую `SeedRng` и конфигурацию.
- Инициализируйте 2D сетку рандомными значениями (например, 45% стен).
- Напишите цикл итераций, где каждая клетка считает количество соседей (Moore neighborhood, 8 соседей).
- Примените правила (B5678/S45678 или стандартные для пещер) для сглаживания стен.

#### Шаг 2: Алгоритм Flood Fill
- Реализуйте алгоритм заливки для поиска всех связных пустых областей (пещер).
- Найдите самую большую область (largest connected component) — это будет играбельная зона.
- Заполните все остальные (меньшие) области стенами, чтобы избежать изолированных комнат.

#### Шаг 3: Интеграция генератора (src/gen/design_floors/collapsed_sector.ts)
- Напишите главную функцию генерации этажа `generateCollapsedSector`.
- Преобразуйте выход `Uint8Array` из автоматов в формат `world.cells`.
- Назначьте текстуры: вместо обычных стен используйте `Tex.CONCRETE`, `Tex.ROTTEN` и фрагменты `Tex.F_CONCRETE`. Не используйте природные текстуры камня.

#### Шаг 4: Размещение труб и коммуникаций
- Пройдитесь по периметру сгенерированных стен. С небольшим шансом (например, 5%) помечайте кластеры стен тегом `'exposed_pipes'` (для последующей системы разрушаемости).
- Расставьте `Feature.RUBBLE` или другие визуальные элементы разрушения на полу.

#### Шаг 5: Регистрация в манифесте (src/data/design_floors.ts)
- Выделите Z-индекс для обрушенного сектора.
- Добавьте вызов `generateCollapsedSector` в манифест этажей или свитчер `main.ts` / `floor_manifest.ts`.

#### Шаг 6: Тестирование и Валидация
- Запустите команду `npm run test:generation` (если она доступна) или изолированный тест генератора.
- Проверьте, что карта всегда имеет путь от старта к лифту (одна связная компонента).
