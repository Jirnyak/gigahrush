## Текущая задача
**Отображение радиуса шума при выстрелах**

### Контекст задачи
Механика стелса и агрессии НПЦ частично завязана на шум (Noise), но игрок никак визуально не понимает, насколько громкий звук издает его оружие или шаги. Необходимо добавить радиальное визуальное отображение (Волны звука/Noise Rings) на миникарте или в основном игровом окне.

### Конкретные файлы и паттерны
- **`src/systems/events.ts`**
- **`src/systems/ai/hearing.ts`**
- **`src/render/hud.ts`**
- **`src/render/particles.ts`**

## Детальный план реализации (Шаги для Агента Jules)

### Шаг 1. Генерация событий шума (Noise Events)
Убедитесь, что в `src/systems/events.ts` есть надежное событие `NOISE_EMITTED`.
```typescript
// Заглушка
export interface NoiseEvent {
  type: 'NOISE_EMITTED';
  x: number;
  y: number;
  radius: number;
  sourceId: number;
}
```
Все виды огнестрельного оружия и взрывы должны публиковать это событие.

### Шаг 2. Визуализация на миникарте
В `src/render/hud.ts` (или модуле миникарты) слушайте эти события и рисуйте расширяющиеся круги, которые плавно затухают (fade-out).
```typescript
export function drawNoiseRings(ctx: CanvasRenderingContext2D, world: World) {
  // Отрисовка кругов с учетом радиуса и таймера угасания
}
```

### Шаг 3. Акустические барьеры
Проведите рефакторинг `src/systems/ai/hearing.ts`. Звук не должен проходить сквозь толстые стены так же легко, как по коридору. Введите простое гашение (dampening) радиуса шума при пересечении `BLOCK_WALL` тайлов.
Это можно сделать через Raycast (считая количество стен) или просто захардкодить 50% снижение громкости за стену.

### Шаг 4. Реакция НПЦ (A-Life Alertness)
НПЦ, находящиеся в радиусе услышанного шума, должны переходить из состояния `IDLE` в `ALERTED` и двигаться к координатам источника звука (Investigation state).
Обновите стейт-машину в `ai.md`/`src/systems/ai/core.ts`.

### Шаг 5. Настройки Оружия
В `src/data/weapons.ts` укажите четкий параметр `noiseRadius` для каждого ствола. Пистолет с глушителем - 5 тайлов, дробовик - 30 тайлов.

### Шаг 6. Тестирование видимости и стелса
Напишите тесты в `tests/systems/hearing.test.ts`.
Проверка: НПЦ в соседней закрытой комнате не слышит тихие шаги, но слышит взрыв гранаты.

### Шаг 7. Сборка и PR
Запустите `npm run check:browser` и визуально убедитесь, что круги шума выглядят органично, не просаживают ФПС (очистка мертвых эффектов) и соответствуют стилистике ГИГАХРУЩА.
