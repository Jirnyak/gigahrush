## Текущая задача
**Баг туториала (раковина -> туалет)**

### Контекст задачи
Сейчас питье из раковины мгновенно дает 10% нужды в туалет. Это костыль, который ломает погружение. Игрок выпил воду, и сразу захотел в туалет. Необходимо использовать элегантную систему пищеварения, которая уже заложена в архитектуре: вода попадает в желудок/мочевой пузырь и обрабатывается постепенно со временем (digestion buffer).

### Конкретные файлы и паттерны
- **`src/systems/interactions.ts`** (где сейчас происходит мгновенное начисление)
- **`src/systems/needs.ts`** (где обрабатывается голод, жажда, усталость)
- **`src/core/types.ts`** (поля нужд в `Entity`)

### 2. Изменение взаимодействий (interactions.ts)
- Найдите обработчик раковины (`SINK` или предмет `WATER_BOTTLE`).
- Уберите прямое увеличение `bladder` на 10.
- Вместо этого, питье уменьшает `thirst` и увеличивает `stomachWater`.

```typescript
// В src/systems/interactions.ts или предметах
export function consumeWater(entity: Entity, amount: number) {
    entity.needs.thirst = Math.max(0, entity.needs.thirst - amount);
    entity.needs.stomachWater += amount; // Вода в животе
}
```

### 3. Симуляция метаболизма (needs.ts)
- В функции `updateNeeds(entity, deltaTime)` (которая вызывается раз в N тиков), реализуйте постепенный перенос из желудка в мочевой пузырь.
- Скорость переваривания (digestion rate) задается константой.

```typescript
// src/systems/needs.ts
const DIGESTION_RATE = 0.5; // Единиц в игровой час (пример)

export function updateMetabolism(entity: Entity, dtHours: number) {
    // Переработка воды
    if (entity.needs.stomachWater > 0) {
        const digested = Math.min(entity.needs.stomachWater, DIGESTION_RATE * dtHours);
        entity.needs.stomachWater -= digested;
        entity.needs.bladder += digested; // Вот теперь плавно хочется в туалет
    }
    
    // Аналогично для еды (stomachFood -> bowels)
}
```

### 4. Влияние на статы
