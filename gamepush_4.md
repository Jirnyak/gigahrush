# Замечание 4: Отсутствие кнопки "Продолжить" в главном меню

## Проблема
Из меню нельзя сразу продолжить сохраненную игру — кнопки "Продолжить" нет. Игроку приходится нажимать "Новая игра" и только оттуда загружаться. Это создает у игроков ложное впечатление, что сохранения сломаны.

## План коррекции
1. Добавить UI-элемент (кнопку) "Продолжить" (Continue) на экран главного меню.
2. При загрузке меню читать наличие валидного сохранения через `localStorage` (с использованием `src/systems/save_runtime.ts` / констант версий).
3. Если сохранение найдено — отображать кнопку. Если нет — скрывать.
4. Привязать к кнопке вызов флоу загрузки игрового стейта и старт сцены этажа (минуя логику генерации новой игры).

## Статус
**РЕШЕНО**:
- В `src/main.ts` добавлена проверка наличия сохраненной игры с правильной мажорной версией (`hasValidSaveGame`).
- Главное меню расширено: теперь при наличии подходящего сохранения отображается кнопка "ПРОДОЛЖИТЬ" (`continue`) как первый пункт.
- При нажатии "Продолжить" происходит инициализация движка (`initGame()`) и сразу после этого загрузка сохраненного состояния (`loadGame()`) в рамках одного флоу (`continueGameFromTitle`).
- Строки `setupContinueLabel`, `setupContinueValue`, `setupContinueHint` добавлены в `src/data/languages.ts` для поддержки локализации.
