## Текущая задача
**Рефакторинг системы типов урона (Damage Types)**

### Контекст задачи
На данный момент урон представляет собой просто число. Для развития боевой системы (Fight System) необходимо добавить поддержку различных типов урона: Физический (Physical), Пси-излучение (PSI), Химический/Токсичный (Chemical), Огненный (Fire). Это позволит предметам, аномалиями и броне иметь сопротивления.

### Конкретные файлы и паттерны
- **`src/core/types.ts`**
- **`src/systems/combat.ts`**
- **`src/data/weapons.ts`**
- **`src/data/items.ts`**

## Детальный план реализации (Шаги для Агента Jules)

### Шаг 1. Определение Enum и типов
В `src/core/types.ts` определите перечисление типов урона и расширьте интерфейс `CombatStats`.
```typescript
export enum DamageType {
  PHYSICAL = 'PHYSICAL',
  PSI = 'PSI',
  CHEMICAL = 'CHEMICAL',
  FIRE = 'FIRE'
}

export interface DamageResistance {
  [DamageType.PHYSICAL]: number;
  [DamageType.PSI]: number;
  [DamageType.CHEMICAL]: number;
  [DamageType.FIRE]: number;
}
```

### Шаг 2. Рефакторинг функции нанесения урона
В `src/systems/combat.ts` перепишите `applyDamage(world, target, amount)` на `applyDamage(world, target, amount, type)`.
Реализуйте формулу: `finalDamage = amount * (1 - resistance[type])`.
Убедитесь, что урон не опускается ниже 0.

### Шаг 3. Обновление данных оружия и аномалий
Пройдитесь по `src/data/weapons.ts` и `src/data/psi.ts`, добавив всем атакам соответствующий `DamageType`.
Например, Дубинка -> PHYSICAL, Отрыжка мутанта -> CHEMICAL.
Обновите `src/systems/procedural_anomalies.ts`, чтобы аномалии наносили правильный тип урона.

### Шаг 4. Обновление брони и артефактов
В `src/data/items.ts` добавьте предметам экипировки модификаторы `DamageResistance`.
Свяжите их с системой инвентаря игрока: суммируйте резисты при расчете итоговых статов.

### Шаг 5. Визуальная обратная связь (UI/Particles)
При получении урона определенного типа можно красить цифры урона (Floating Damage) в разные цвета (PSI - фиолетовый, CHEMICAL - зеленый, FIRE - оранжевый) в `src/render/hud.ts`.

### Шаг 6. Тестирование (Unit & Smoke)
