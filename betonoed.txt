Бетоноед / BETONOED: кодовая справка по текущему рабочему дереву
==================================================================

Сводка
------

Бетоноед - отдельный monster package и авторская встреча в Коллекторах
(FloorLevel.MAINTENANCE). В коде это не обычный случайный монстр из весов:
его ecology entry имеет маленький spawnWeight: 0.12, а главная авторская
встреча создается модулем src/gen/maintenance/betonoed_shortcut.ts.

Игровая роль: слабостенный бетонный монстр. Он создает решение вокруг
подготовленного шва: ждать и рисковать коротким ходом, запечатать стену
герметиком/блок-комплектом, ускорить прогрыз шумом, отвлечь шумом в сторону
или отогнать огнем.

Основные файлы
--------------

- src/core/types.ts
  - enum MonsterKind содержит BETONOED.
  - комментарий enum: weak-wall breacher / бетоноед, прогрызает слабые стены.

- src/entities/betonoed.ts
  - главный пакет монстра: DEF и generateSprite().

- src/entities/monster.ts
  - регистрирует DEF в MONSTERS.
  - регистрирует generateSprite в MONSTER_SPRITES.
  - включает BETONOED в NEW_MONSTER_KINDS.
  - включает BETONOED в NEW_MONSTERS_BY_FLOOR[FloorLevel.MAINTENANCE].

- src/data/monster_ecology.ts
  - ecology card: роль, cue, правило, этажи, комнаты, rare, loot, rumors.
  - spawnWeight: 0.12, то есть редкий весовой спавн плюс авторская сцена.

- src/gen/maintenance/betonoed_shortcut.ts
  - авторская сцена "Бетоноед: шумная кладовая у слабой стены".
  - строит комнаты, слабую стену, предметы контрплея, спавнит Бетоноеда.
  - регистрирует interaction hook для E.
  - регистрирует runtime hook в фазе pre_ai.

- src/gen/maintenance/content_manifest.ts
  - подключает generateBetonoedShortcut() как maintenance content.
  - POI metadata id: maint_betonoed_shortcut.
  - debugLabel: "Коллекторы: слабая стена Бетоноеда".

- src/systems/ai/monster.ts
  - общий AI читает aiFlags.
  - wallBias дает бонус у стены/угла и readability events.
  - BETONOED получает отдельные тексты wall-bias cue/open-break.
  - weakWallBreach как отдельная generic AI-ветка сейчас не используется;
    сам прогрыз реализован модулем betonoed_shortcut.ts.

- src/render/sprite_index.ts
  - sprite index считается автоматически через enum/registry.
  - spawn-код не хранит магическое число спрайта.

- src/render/sprites.ts
  - generateSprites() вызывает MONSTER_SPRITES[kind]().
  - после построения sheet проставляет def.sprite = monsterSpr(kind).

- src/entities/procedural_visuals.ts
  - для всех монстров возможен per-entity procedural sprite.
  - для BETONOED задана своя палитра tint и темные трещинные marks.

- src/render/webgl.ts
  - при рисовании item drops и procedural actors могут подменять shared atlas.
  - для монстров WebGL сначала пытается взять proceduralEntityTexture(e),
    потом fallback на общий sprite sheet.

- src/data/rumors.ts, src/data/tips.ts, src/systems/world_log.ts
  - слухи, подсказки и логовые формулировки про слабую стену, прогрыз,
    запечатывание, короткий ход и отгон огнем.

- tests/monster_42_betonoed.test.ts
  - проверяет standalone package, registry wiring, rare positive ecology weight,
    отсутствие старого variant plumbing и прямой MonsterKind.BETONOED.

Как он попадает в игру
----------------------

Базовый реестр позволяет иметь MonsterKind.BETONOED на этаже Maintenance.
ecology spawnWeight равен 0.12, поэтому он может попасть в generic
maintenance-подбор как редкая угроза. Главный ручной путь появления остается
авторским: generateBetonoedShortcut(ctx) в
src/gen/maintenance/betonoed_shortcut.ts.

Подключение:

- src/gen/maintenance/content_manifest.ts импортирует generateBetonoedShortcut.
- В manifest модуль обернут в withPoiGenerationMetadata().
- Metadata:
  - id: maint_betonoed_shortcut
  - floor: maintenance
  - debugLabel: Коллекторы: слабая стена Бетоноеда
  - decisionHooks:
    - repair / betonoed_seal_weak_wall
    - flee / betonoed_noise_lure
    - kill / betonoed_fire_drive_off
    - reroute / betonoed_shortcut_used

Генерация сцены:

- Ищет maintenance area рядом со spawn:
  - размер поиска для POI: 32 x 18
  - дистанция от spawn: 85..170
- Создает две комнаты:
  - storage 13 x 9:
    "Бетоноед: шумная кладовая у слабой стены"
  - corridor 13 x 9:
    "Бетоноед: короткий ход за съедаемой стеной"
- Между ними ставит weak cell:
  - Cell.WALL
  - Tex.CONCRETE
  - Feature.NONE
  - aptMask = 0
  - hermoWall = 0 при первичной генерации
  - roomMap = -1
- Рядом красит полы в Tex.F_CONCRETE.
- Ставит features:
  - в кладовой: SHELF, SCREEN, LAMP, APPARATUS рядом со швом;
  - в коротком ходе: CANDLE, MACHINE, SHELF.
- Ставит surface splat/пыль у слабой стены через stampSurfaceSplat().
- Кладет предметы в кладовую:
  - noise_can
  - bottled_voice
  - sealant_tube
  - block_kit
  - flamethrower
  - ammo_fuel
- Кладет предметы в shortcut room:
  - rebar
  - ammo_fuel
  - psi_concrete_splinter

spawnBetonoed():

- Берет def = MONSTERS[MonsterKind.BETONOED].
- Ставит монстра в floor cell внутри bait room, предпочтительно weakX - 2,
  weakY.
- hp рассчитывается от zoneLevel:
  hp = round(def.hp * (0.74 + zoneLevel * 0.04))
  При zoneLevel 4 это 324 hp.
- speed = def.speed, без local scaling.
- sprite = monsterSpr(MonsterKind.BETONOED).
- name = "Бетоноед".
- monsterKind = MonsterKind.BETONOED.
- ai goal: WANDER, target weakX/weakY.
- rpg = randomRPG(zoneLevel).


Runtime state встречи
---------------------

Источник: src/gen/maintenance/betonoed_shortcut.ts.

Встреча хранит module-local activeBetonoed: BetonoedState | null.
Это не отдельная save/load секция. Поиск по коду не нашел прямой betonoed-save
payload. Состояния сцены живут в этом runtime object плюс в изменениях World,
entities и events.

BetonoedState поля:

- weakIdx
- weakX
- weakY
- nearRoomId
- farRoomId
- monsterId
- breached
- sealed
- used
- drivenOff
- warned
- noiseBaitSeen
- breachAt
- scanAt
- fireExposure
- lastHp
- damageNoticeAt
- lastNoiseId
- lureNoticeAt

Runtime constants:

- NOISE_BAIT_IDS:
  - noise_can
  - radio
  - bottled_voice
  - siren_shard
- BREACH_NEAR_SQ = 16 * 16
- BREACH_KEEPALIVE_SQ = 24 * 24
- NOISE_BAIT_SQ = 5 * 5
- NOISE_LURE_SQ = 9 * 9
- FIRE_DRIVE_OFF_SECONDS = 1.1

Хуки:

- registerContentInteractionHook({ id: "maint_betonoed_shortcut", use(...) })
  вызывает tryUseBetonoedShortcut().
- registerContentRuntimeHook({ id: "maint_betonoed_shortcut", phases: ["pre_ai"] })
  вызывает updateBetonoedShortcut().
- main.ts вызывает updateContentRuntimeHooks(... phase: "pre_ai" ...) перед
  обычным updateAI().
- Если hook возвращает worldChanged, main.ts вызывает updateWorldData(world).


Игровая логика слабой стены
---------------------------

tryUseBetonoedShortcut(world, player, state, lookX, lookY):

- Работает только если:
  - activeBetonoed есть;
  - state.currentFloor === FloorLevel.MAINTENANCE;
  - look point игрока находится рядом со слабой стеной:
    world.dist2(lookX, lookY, weakX + 0.5, weakY + 0.5) <= 2.25.
- Если шов уже sealed:
  - сообщение: "Слабая стена уже запечатана."
- Если у игрока есть sealant_tube:
  - removeItem(player, "sealant_tube", 1)
  - sealWeakWall(..., "sealant")
- Если у игрока есть block_kit:
  - removeItem(player, "block_kit", 1)
  - sealWeakWall(..., "block_kit")
- Если предмета нет:
  - если проход breached:
    "Открытый прогрыз можно закрыть комплектом блока или герметиком."
  - иначе:
    "Слабый шов. Герметик запечатает его, шумовая приманка ускорит прогрыз,
    огонь отгонит Бетоноеда."

setWeakWall(world, encounter, wall):

- wall = true:
  - world.cells[weakIdx] = Cell.WALL
  - hermoWall[weakIdx] = 1
  - wallTex[weakIdx] = Tex.CONCRETE
- wall = false:
  - world.cells[weakIdx] = Cell.FLOOR
  - floorTex[weakIdx] = Tex.F_CONCRETE
- always:
  - features[weakIdx] = Feature.NONE
  - aptMask[weakIdx] = 0
  - roomMap[weakIdx] = -1
  - markWallTexDirty()
  - markFloorTexDirty()

sealWeakWall():

- Если еще не sealed:
  - ставит wall=true;
  - sealed = true;
  - breachAt = 0;
  - пишет сообщение:
    - если уже breached:
      "Проход запечатан. Бетоноед скребёт бетон с другой стороны."
    - если до прогрыза:
      "Слабый шов запечатан. Бетоноед потерял короткий выход."
  - publishEvent type door_sealed:
    - tags: maintenance, betonoed, shortcut, sealed, reason
    - itemId: sealant_tube или block_kit
    - data.system: betonoed_shortcut

breachWall():

- Если не breached и не sealed:
  - setWeakWall(..., false)
  - breached = true
  - breachAt = 0
  - сообщение:
    "Бетоноед прогрыз бетон. Открылся короткий ход, но он не пустой."
  - publishEvent type door_opened:
    - tags: maintenance, betonoed, shortcut, wall_breached, reason
    - reason: noise или proximity
    - targetName: "слабая бетонная стена"
    - monsterKind: MonsterKind.BETONOED
    - severity: 5

maybeMarkShortcutUsed():

- Если игрок вошел в farRoomId после breached и до sealed:
  - used = true
  - сообщение:
    "Вы проскочили через ход Бетоноеда. Маршрут стал короче, воздух - хуже."
  - publishEvent type door_opened:
    - tags: maintenance, betonoed, shortcut, shortcut_used, player


Как он начинает прогрыз
-----------------------

updateBetonoedShortcut() работает в pre_ai.

Проксимити:

- Если игрок рядом со слабой стеной:
  world.dist2(player.x, player.y, weakX + 0.5, weakY + 0.5) <= 16^2
- И breachAt еще не поставлен:
  breachAt = state.time + 2.8
- Сообщение-предупреждение:
  "За слабой стеной хрустит бетон. Герметик, блок-комплект, шумовая приманка
  или огонь решат исход."

Если игрок ушел дальше keepalive radius 24 клеток до прогрыза и не было noise
bait:

- breachAt = 0
- warned = false
- сообщение:
  "Стена затихла. Бетоноед потерял интерес к этому шву."

Шумовая приманка как item drop:

- Каждые 0.45 секунды сканируется область около weak wall.
- noiseBaitNear() ищет ITEM_DROP с inventory из NOISE_BAIT_IDS.
- Радиус приманки у слабой стены: 5 клеток.
- Если bait найден и игрок рядом с encounter в радиусе 24:
  - noiseBaitSeen = true
  - breachAt = min(existing breachAt или Infinity, state.time + 0.8)
  - warnBreach(..., "noise")
  - сообщение:
    "Бетоноед пошёл на шум. Слабая стена скоро станет проходом."

Шумовая приманка как live noise:

- applyNoiseLure() использует findNoiseForActor() с:
  - minSeverity: 2
  - scanInterval: 0.35
  - hearingMult: 1.35
- Если noise рядом со слабой стеной в радиусе 9:
  - noiseBaitSeen = true
  - breachAt = min(existing breachAt или Infinity, state.time + 0.8)
  - monster.ai.goal = AIGoal.GOTO к weakX/weakY
  - warnBreach(..., "noise")
- Если noise не у шва:
  - если breachAt > 0, таймер прогрыза отодвигается минимум до state.time + 1.8
  - monster.ai.goal = AIGoal.GOTO к точке шума
  - раз в 2.8 секунды сообщение:
    "Бетоноед ушёл на шум. Слабая стена получила короткую паузу."

Урон по Бетоноеду до прогрыза:

- Если hp уменьшился до breached/sealed:
  - если breachAt > 0, таймер отодвигается минимум до state.time + 1.4
  - раз в 2.4 секунды сообщение:
    "Бетоноед сбился с прогрыза и отступил от шва."

Огонь:

- flameNear() ищет PROJECTILE с projType === ProjType.FLAME рядом с монстром.
- Радиус огня: 3.2 клетки.
- Пока flame рядом:
  - fireExposure += dt
  - monster.speed = max(0.25, monster.speed * 0.985)
- Если exposure >= 1.1 секунды:
  - driveOffBetonoed(..., "fire")
  - drivenOff = true
  - monster.alive = false
  - monster.hp = 0
  - сообщение:
    "Огонь выгнал Бетоноеда в трещину. Проход остался, тварь ушла."
  - publishEvent type death_seen:
    - tags: maintenance, betonoed, shortcut, driven_off, fire
- Если огня нет:
  - fireExposure спадает на dt * 0.5.


Обычный бой и AI
----------------

Бетоноед - melee/contact monster, не ranged:

- isRanged не задан.
- proj fields не заданы.
- Основные боевые числа:
  - hp 360 в DEF, но authored spawn пересчитывает hp по zoneLevel.
  - speed 0.98.
  - dmg 28.
  - attackRate 2.35.

Generic AI flags:

- wallBias:
  - общий AI проверяет wallTerrainPressureActive().
  - давление активно, если сам монстр рядом со стеной/узким углом или цель
    рядом со стеной/узким углом.
  - movement multiplier:
    - 1.18 рядом со стеной или в narrow door/corner;
    - 0.92 на открытом месте.
  - damage multiplier:
    - 1.2 если wallTerrainPressureActive().
  - scan cooldown для wallBias монстров: 1.1 секунды.
  - readability cue cooldown: 5.2 секунды.
  - когда wallBias активен, spriteScale поднимается минимум до 1.03.
  - когда преимущество стены сломано открытым полом, spriteScale падает до 0.96.

Специальные тексты wallBias для BETONOED:

- active cue:
  "Бетоноед ведет челюсть вдоль бетонного шва. Шум, огонь или герметик решают
  темп."
- open-floor break:
  "Бетоноед потерял бетонный шов и сбился с короткого выхода."
- readability event tags:
  - betonoed
  - wall_bias или open_floor/wall_advantage_broken

weakWallBreach:

- Есть в типе MonsterAIFlag.
- Есть в DEF.aiFlags.
- Проверяется tests/monster_42_betonoed.test.ts.
- Отдельного generic AI branch по этому флагу в текущем src/systems/ai/monster.ts
  не найдено. Фактический breach/shortcut behavior находится в
  src/gen/maintenance/betonoed_shortcut.ts.

Спрайт: базовый генератор
-------------------------

Источник: src/entities/betonoed.ts.

generateSprite() возвращает Uint32Array размером S * S, где S = TEX = 64.
Пиксели packed RGBA little-endian через rgba() из src/render/pixutil.ts.
CLEAR - прозрачный пиксель.

Алгоритм:

1. Создает прозрачный массив 64 x 64.
2. Рисует приземистое пыльное тело:
   - y 18..54
   - центр cx = 32
   - ширина зависит от sin-bulge;
   - эллиптическая маска;
   - шум noise(x * 2, y * 2, 12420) добавляет пыльную вариативность;
   - chip noise(x * 5, y * 4, 12421) иногда затемняет сколы;
   - базовые цвета около RGB 142/138/124.
3. Рисует круговую grinder-mouth:
   - центр mx = 33, my = 35;
   - эллипс примерно 24 x 22;
   - внешний красный gum ring RGB 150/34/28;
   - зубная зона с шумом:
     - зубы RGB 214/206/176;
     - темная ткань/камень RGB 84/68/58;
   - горло RGB 18/12/12.
4. Рисует chalk-dust forearms:
   - y 36..57;
   - две боковые руки/следа около cx - 16 и cx + 16;
   - alpha убывает по step.
5. Добавляет aggregate speckles / powder plume:
   - 95 попыток;
   - если пиксель уже skin, ставит pebble цвета:
     - 92/86/76
     - 184/178/150
     - 112/104/92
   - если пустое место вокруг тела, иногда ставит пыль RGB 214/210/190 alpha 95.
6. Добавляет маленькие красные gum glints:
   - точки около рта на my - 8 и my + 8;
   - ярко-красные и темно-красные пиксели.

Метрики generateSprite(), снятые через npx tsx:

- S: 64
- opaquePixels: 1223
- bounds:
  - minX: 9
  - minY: 18
  - maxX: 53
  - maxY: 58
  - width: 45
  - height: 41
- avgAlpha: 236
- sprite hash FNV-style из тестового скрипта: 945446253

Важно:

- DEF.sprite сначала 0.
- generateSprites() в src/render/sprites.ts строит общий sheet и потом
  проставляет def.sprite = monsterSpr(kind).
- spawnBetonoed() не ждет def.sprite и прямо использует
  monsterSpr(MonsterKind.BETONOED).


Спрайт: per-entity procedural visuals
-------------------------------------

Источник: src/entities/procedural_visuals.ts и src/render/webgl.ts.

Для всех EntityType.MONSTER:

- entityUsesProceduralSprite(e) возвращает true.
- proceduralEntitySpriteKey(e) строит seed/key из:
  - spriteSeed или hash от id/x/y/name/plotNpcId;
  - e.type;
  - e.monsterKind;
  - occupation;
  - e.sprite;
  - npcVisualId/isFemale/faction, если есть.
- generateProceduralEntitySprite(e) для монстра вызывает
  generateProceduralMonsterSprite(e.monsterKind, seed, ...).

Для BETONOED:

- специальных unique branches нет, как у NIGHTMARE/ROBOT/IDOL и т.п.
- используется базовый MONSTER_SPRITES[MonsterKind.BETONOED]().
- затем mutateMonsterSprite() накладывает tint:
  - палитры BETONOED:
    - [168, 160, 132]
    - [138, 132, 116]
    - [194, 178, 146]
  - strength = 0.18..0.46 по seed.
  - шум и vein добавляют локальные вариации.
  - иногда pock убирает отдельные пиксели.
- addMonsterMarks() для бетонных/арматурных монстров использует темный mark:
  - [40, 34, 30]
  - рисует 2..6 линий-трещин по seed.

WebGL:

- draw path сначала пробует itemDropTexture(e) для item drops.
- потом proceduralEntityTexture(e).
- если procedural texture нет, fallback на shared spriteTextures[sprIdx].
- procedural texture кэшируется в glState.proceduralSpriteTextures.


Контрпредметы
-------------

Прямо в betonoed_shortcut:

- sealant_tube:
  - если игрок смотрит на слабую стену и предмет есть, тюбик расходуется,
    wall=true, sealed=true.
- block_kit:
  - аналогично sealant_tube, расходуется 1 штука.

  - если лежат как ITEM_DROP рядом со слабой стеной, ускоряют прогрыз до 0.8 с.
  - если live noise не у стены, могут увести монстра от шва и дать паузу.
- flamethrower / flame projectile:
  - ProjType.FLAME рядом с монстром 1.1 секунды отгоняет/убивает его
    через driveOffBetonoed().
- ammo_fuel:
  - кладется рядом как топливо для огнемета.

Предметные определения:

- sealant_tube: "Тюбик герметика", misc, дешевый состав для щелей.
- block_kit: tool, durability 1, устанавливает один блок стены.
- noise_can: misc, stack 6, noise/decoy/counterplay.
- flamethrower: weapon, projType FLAME, ammoType ammo_fuel.
- ammo_fuel: ammo для огнемета.
- bottled_voice и siren_shard входят в NOISE_BAIT_IDS сцены.

Пределы и важные замечания
--------------------------

- Это не новый FloorLevel и не отдельная подсистема.
- Это MonsterKind + data package + authored Maintenance content module.
- Случайная ecology generation может выбрать Бетоноеда редко из-за
  spawnWeight: 0.12.
- Прогрызает не любые стены, а только подготовленный weak cell в authored POI.
- generic AI flag weakWallBreach сейчас не имеет отдельного runtime branch;
  фактический topology change делает betonoed_shortcut.ts.
- Runtime logic bounded:
  - нет full-world scan;
  - item bait scan идет по entities, но только раз в 0.45 с и проверяет
    дистанцию до одной weak wall;
  - noise sense использует findNoiseForActor() с scanInterval 0.35;
  - все решения привязаны к одной активной encounter state.
- События compact/id-based:
  - roomId, zoneId, weak cell, monsterKind, tags, reason.
- Отдельной save/load shape для activeBetonoed нет.
- Код не импортирует Бетоноеда в main.ts напрямую; main.ts видит только общий
  content hook result и worldChanged.
