## Текущая задача
**Механики: Универсальная разрушаемость декора (столы, бочки, фичи).**



### Контекст задачи
ПОЛНАЯ разрушаемость объектов окружения. ВСЕ Feature-ы разрушаемы ударами, кроме `Feature.LIFT_BUTTON` (plot-critical) и `Feature.SCREEN` (терминалы). Даже кровати, станки и аппараты — ломай всё.

### Конкретные файлы и паттерны
- **`src/core/types.ts`**: `Feature` enum — TABLE, CHAIR, BED, SHELF и т.д. При разрушении: `world.features[idx] = Feature.NONE`.
- **`src/systems/interactions.ts`**: При ударе по ячейке с Feature — уменьшить HP (sparse map `featureHp: Map<number, number>`).
- **Дроп-таблица по Feature type**:
  - `TABLE` → `'wood_plank'` (80%) или `'metal_scrap'` (20%)
  - `CHAIR` → `'wood_plank'` (90%)
  - `BED` → `'cloth_rag'` (60%), `'spring'` (30%)
  - `SHELF` → `'wood_plank'` (50%), случайный предмет из room loot pool (10%)
  - `MACHINE` → `'metal_scrap'` × 2 (70%), `'electronics'` (20%)
  - `APPARATUS` → `'electronics'` (50%), `'wire'` (40%)
  - `STOVE` → `'metal_scrap'` (80%)
  - `SINK` → `'pipe_fragment'` (70%)
  - `TOILET` → `'pipe_fragment'` (60%), splash effect
  - `LAMP` → тьма в комнате + `'glass_shard'` (90%). ОБНОВИТЬ `world.light` через dirty flag!
  - `CANDLE` → ничего, но тьма
  - `DESK` → `'wood_plank'`, иногда документ из комнаты
- **Version bumps**: `world.surfaceVersion++` при удалении фичи.
- **Path blockers**: Если фича была в `World.pathBlockers` — обновить blocker (убрать запись).
- **`src/render/` mesh pass**: Визуальные слоты мешей привязаны к фичам. При удалении фичи — меш тоже пропадает автоматически.

### Детальный Пошаговый План Реализации (Workflow)

#### Шаг 1: Добавление HP для фич (src/core/world.ts)
- Аналогично стенам, добавьте `featureHp: Map<number, number>` в стейт мира.
- Определите базовое ХП для каждого типа `Feature` в отдельном конфиге.
- Не забудьте исключить критичные сюжетные фичи (кнопка лифта, экраны терминалов).

#### Шаг 2: Обработка урона по фичам (src/systems/interactions.ts)
- Модифицируйте melee-атаки: если атака попадает в ячейку с фичей (даже если это оружие, а не инструмент), наносите урон фиче. Обычное оружие наносит `meleeDmg`, инструменты могут наносить `wallDmg`.
- При `HP <= 0` переходите к фазе разрушения.

#### Шаг 3: Логика дропа и очистки (src/systems/destructibility.ts)
- Разрушение фичи:
  1. `world.features[cellIdx] = Feature.NONE`.
  2. Удалите индекс из `world.pathBlockers`, если он там был (например, станки).
  3. `world.surfaceVersion++`.
- Система дропа: вызовите функцию, которая по типу фичи спавнит предметы в инвентарь или на пол (spawnItemOnFloor). Реализуйте шансы (Math.random() или SeedRng).

#### Шаг 4: Обновление освещения
- ОСОБЫЙ СЛУЧАЙ: Если разрушена `Feature.LAMP`, удалите источник света из массива `world.lights` или аналогичной структуры освещения.
- Инкрементируйте флаг грязного света `world.lightVersion++`, чтобы рендерер пересчитал тени.

#### Шаг 5: Сохранение и загрузка
- Добавьте `featureHp` в `src/systems/save_runtime.ts`. Поскольку стейт фич хранится в `world.features`, удаленные фичи загрузятся как `NONE` автоматически, но полу-разрушенные фичи требуют сохранения их текущего HP.

#### Шаг 6: Тестирование
- Напишите тесты: сломать лампу -> проверить `lightVersion`, сломать стол -> проверить дроп досок.
