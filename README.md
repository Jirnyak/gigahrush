# ГИГАХРУЩ

**Бесконечный бетонный лабиринт — survival horror life-sim в процедурно-генерируемой гигаструктуре.**

 Zero dependencies. Процедурные текстуры, спрайты, звук. 

## Концепция

Тороидальный мир 1024×1024 клеток. Бесконечная бетонная хрущёвка без выхода. Три этажа: **Жилая зона**, **Коллекторы** (трубные туннели) и **Ад** (мясная преисподняя). Комнаты — квартиры, кухни, санузлы, склады, медпункты, цеха, залы, курилки, бухгалтерии — соединены коридорами. Мир разделён на **64 зоны** с фракционным контролем; зоны постепенно захватываются **фиолетовым туманом** самосбора.

Жители (до 1024 NPC) живут семьями в квартирах по **24-часовому расписанию**: просыпаются, завтракают, идут на работу, обедают, отдыхают, спят. По лабиринту бродят **путники** (граждане), **паломники** (культисты) и **охотники** (ликвидаторы) — автономные NPC-странники, не привязанные к квартирам. Каждый NPC управляется **конечным автоматом** (A-Life FSM) и занимает рабочее место в зависимости от профессии. Периодически происходит **САМОСБОР** — из стен вылезают монстры, двери герметично запираются, фиолетовый туман поглощает очередную зону. После окончания самосбора лабиринт перестраивает сам себя: коридоры разрушаются и отрастают заново.

Игрок начинает в **Актовом зале** — брифинг-комнате со школьными партами и пропагандистскими слайдами на стене. Выживайте.

## Системы

### Три этажа (FloorLevel)

| Этаж | Название | Описание |
|------|----------|----------|
| 0 | **Жилая зона** | Квартиры, мастерские, залы. 128 квартир-кластеров, 1024 NPC |
| 1 | **Коллекторы** | Плотная сеть трубных туннелей. Стены из труб, водяные каналы. Монстры: сборки + ползуны |
| 2 | **Ад** | Мясные стены и полы. 80 культистов-паломников + 80 усиленных монстров. Ускоренный самосбор |

Переход между этажами через **лифты** (`Cell.LIFT` + `Feature.LIFT_BUTTON`). При переключении мир перегенерируется, инвентарь и ХП сохраняются.

### Стартовый атриум (Актовый зал + Оружейная)

Игрок спавнится в **Актовом зале** — комната 11×9 в центре мира. Школьные парты в три ряда, на передней стене — **2 плитки-слайда** с циклической презентацией. **Ольга Дмитриевна** (врач, тьютор) стоит у доски.

Рядом через проход — **Оружейная** (7×14), она же стрельбище. Металлические стены, стойки-барьеры (спрайты парт), **мишени** (текстура `Tex.TARGET`) на дальней стене. **Барни** (ликвидатор, инструктор по оружию) стоит у стоек. На полу лежат патроны 9мм.

Далее в мире — **Лаборатория** Якова Давидовича (отдельный контент-модуль `yakov_lab.ts`). Генерируется на расстоянии **40–100 клеток** от спавна — в существующем медпункте или как новая 7×7 комната. Белая плитка, аппаратура, полки с книгами. **Яков Давидович** (учёный, исследователь ПСИ) стоит у аппаратуры. Путь до него — полноценный геймплей, а не тьюториал.

Стрельбище — не квест, а свободный геймплей: можно стрелять по мишеням, **следы пуль** остаются на стенах (декали в движке).

**Цепочка сюжетных квестов (основной сюжет):**

Линейная цепочка квестов — основная сюжетная линия игры. Каждый квест открывается после завершения предыдущего.

| # | Квестодатель | Задание | Награда | XP |
|---|---|---|---|---|
| 1 | Ольга Дмитриевна | «Сходите к Барни в оружейную» (TALK) | Макаров + патроны 9мм | 10 |
| 2 | Барни | «Доложите Ольге Дмитриевне» (TALK) | Бинты + вода + хлеб | 10 |
| 3 | Ольга Дмитриевна | «Зайдите к Якову Давидовичу» (TALK) | Сгусток: Пси удар | 20 |
| 4 | Яков Давидович | «Культы Чернобога — найдите идол» (FETCH) | Антидепрессант + таблетки + 50₽ | 50 |
| 5+ | *в разработке* | *расширение сюжетной цепочки* | — | — |

**Яков Давидович** — учёный-исследователь пси-явлений хруща. Находится в **Лаборатории** — отдельный контент-модуль (`yakov_lab.ts`), генерируется на расстоянии от спавна. Изучает культистов и их связь с самосбором.

**Идол Чернобога** — тёмная каменная фигурка культистов. 128 штук разбросаны по всему миру в залах, кладовых, бухгалтериях и курилках.

**Маркеры квестов:** активные квесты из журнала Q отображаются на миникарте и полной карте голубыми ромбами (◆). Для TALK-квестов маркер указывает на целевого NPC, для VISIT — на целевую комнату, для FETCH — на квестодателя.

> **Для разработчиков:** сюжетные квесты добавляются в `src/systems/quests.ts` → `generateQuest()`, в секции `isTutor`/`isTutorBarni`/`isTutorYakov`. Новый сюжетный NPC: добавить `isTutorXxx` флаг в Entity (types.ts), **создать отдельный контент-модуль** `src/gen/living/my_npc.ts` (по образцу `yakov_lab.ts` / `start_room.ts`), подключить одной строкой в `index.ts`, добавить ветку в `generateQuest()`. **Каждый сюжетный NPC — отдельный `.ts` файл.** Не мешать в `npcs.ts` или `start_room.ts`. Каждый шаг проверяет `state.quests` на завершение предыдущего квеста — простая линейная цепочка.

**8 слайдов** (4 пары) циклически сменяются каждые 5 секунд:
1. Приветствие: «Товарищи жильцы!» + «Добро пожаловать в блок №XXX»
2. Девиз: «Пусть усердный коллективный труд…» + «…приведёт нас к процветанию и безопасности»
3. Инструкции 1-2: при сирене — к гермодвери; не покидать ячейки до конца самосбора
4. Инструкция 3: увидев фиолетовый туман — немедленно к ближайшему шлюзу!

Комнаты атриума защищены `aptMask` — не разрушаются при самосборе.

### Следы пуль (Bullet Hole Decals)

Выстрелы оставляют **следы на стенах** — темные круглые отметины в месте попадания снаряда. Реализовано шейдерно в рендере: `world.decals` хранит координаты попаданий пер клетку (до 20), а `engine.ts` при отрисовке стены проверяет расстояние каждого текселя до каждого декаля и затемняет попавшие пиксели (radius 3px, darken 0.2).

### 64 зоны и фракционный контроль

Мир разделён на **64 макро-зоны** (сетка 8×8, ~128 клеток на зону). Каждой зоне назначена **фракция** по распределению Вороного: 40% Граждане, 25% Ликвидаторы, 25% Культисты. Зоны отображаются на миникарте уникальными цветами (золотой угол HSL). На границах зон размещаются **шлюзы** (airlock: дверь—пол—дверь).

Во время самосбора случайная зона **захватывается фиолетовым туманом** → её фракция навсегда становится `SAMOSBOR`. Захваченные зоны не восстанавливаются.

### Фиолетовый туман (Purple Fog)

- При самосборе туман засевается в центре захваченной зоны (радиус ~5)
- BFS-расширение каждые 0.15с (30% шанс на клетку)
- **Двери блокируют туман** — герметичные квартиры безопасны
- В туманных коридорах (fog > 100) каждые 2с спавнятся монстры (до 30 + 5 × кол-во самосборов)
- Туман визуализируется на миникарте фиолетовым оверлеем

### Процедурная генерация — Жилая зона (living/)

Алгоритм **room-first + percolation** — **BLAME!**-инспирированная гигаструктура:

1. **128 квартир-кластеров** на сетке 16×16 — 5 типов планировки (Classic 35%, Communal 20%, Studio 15%, Extended 15%, Barrack 15%). Защищены `aptMask` — постоянные, не разрушаются при самосборе.
2. **Актовый зал** — стартовая комната в центре мира, тоже защищена `aptMask`.
3. **64 зоны** — Вороново распределение фракций.
4. **Volatile maze** — заполнение оставшегося пространства:
   - MST-коридоры (Prim) + 12% случайных петель
   - Dense fill — скан каждые 6 клеток, до 3 попыток на точку
   - Локальное подключение fill-комнат
   - **Перколяция** — BFS гарантирует единый связный граф (до 4 проходов)
   - **8 форм комнат** — прямоугольник, эллипс, ромб, крест, полукруг, гексагон, капсула, шеврон
   - Шлюзы на границах зон, лифты между этажами
   - Auto-doors (wall-door-wall паттерн)
   - Декорации: колонны, пилоны, перегородки, сетки
   - Abyss pits — провалы в массивах стен
5. **1024 NPC** — 8 на квартиру + 80 странников (32 путника, 24 паломника, 24 охотника)

При самосборе volatile maze **полностью перестраивается** (`wipeVolatile()` + `generateVolatileMaze()`). Квартиры и актовый зал неприкосновенны.

### Генерация — Коллекторы (maintenance.ts)

- Плотная сеть туннелей: горизонтальные/вертикальные линии каждые 4 клетки, стены `Tex.PIPE`
- Коллекторные комнаты (`"Коллектор #N"`) на сетке 64 клетки (70% шанс)
- **30 водяных каналов** (`Cell.WATER`, ширина 1–3, длина 40–200)
- 4 лифта, разреженное освещение (1.5%), 30 монстров (сборки + ползуны)

### Генерация — Ад (hell.ts)

- Плотные органические комнаты на сетке 8 клеток (65% шанс), стены `Tex.MEAT`, полы `Tex.F_MEAT`
- MST-коридоры, все в мясной текстуре
- 3 лифта, минимальное освещение (0.8%)
- **80 культистов**-паломников (120 ХП, скорость 1.5, ~40% вооружены Сгустками) + **80 усиленных монстров** (TVAR 80hp, POLZUN 130hp, BETONNIK 250hp)
- Ускоренный самосбор (интервал 30–60с вместо 120–180с)

### Мир

- **1024×1024 тороидальный грид** — клетки оборачиваются по обеим осям (тор)
- **Клетки:** FLOOR, WALL, DOOR, ABYSS, LIFT, WATER — каждая может иметь **features** (лампа, стол, стул, кровать, плита, раковина, унитаз, полка, станок, аппарат, парта, слайд, кнопка лифта)
- **Комнаты:** 10 типов (жилая, кухня, санузел, кладовая, медпункт, зал, цех, коридор, курилка, бухгалтерия)
- **128 квартир-кластеров** (5 типов планировок), доп. комнаты в оставшихся секторах
- **Двери:** обычные, запертые (ключ), герметичные (самосбор) — 5 состояний (open/closed/locked/hermetic_open/hermetic_closed)

### Освещение

- **Лампы** — feature на клетке, радиус 8 клеток, тёплое свечение на потолке
- **Lightmap** — Float32Array 1024×1024, запекается из позиций ламп
- **Ambient 0.12** — без лампы почти полная темнота

### Рендер

- **DDA raycaster** (Wolfenstein-стиль) — 320×200 внутреннее разрешение
- **32 процедурные текстуры** 64×64 (бетон, кирпич, панели, плитка, металл, гниль, шторы, линолеум, паркет, ковёр, потолок, двери, бездна, лифт, трубы, вода, мясо, парта, мишень, 8 слайдов)
- **32 процедурных спрайта** (13 NPC по профессиям, 3 странника, 1 дроп, 10 монстров, 1 снаряд глаза, 1 парта, 3 снаряда оружия)
- **Fog** — плотность 0.065 (0.09 при самосборе), glitch-эффект при самосборе
- **Pixel-perfect upscale** на полный экран

### Игровое время (GameClock)

24-часовой цикл:
- **1 игровой час = 60 реальных секунд** (1 игровая минута = 1 реальная секунда)
- Часы отображаются в HUD (нижняя панель + инвентарь)
- Игра начинается в **08:00**
- Полный игровой день = 24 реальных минуты
- Время управляет расписанием NPC, освещением, событиями

### A-Life — система жизни NPC

Каждый NPC живёт по расписанию, управляемому **конечным автоматом** (Finite State Machine). Текущее состояние определяется игровым временем и событиями.

#### NPC состояния (NpcState)

| Состояние  | Часы    | Поведение                                                |
|------------|---------|----------------------------------------------------------|
| SLEEPING   | 22–6    | Спит в своей жилой комнате                               |
| MORNING    | 6–8     | Утренняя рутина: санузел → кухня → коридоры              |
| WORKING    | 8–12    | Идёт на работу (зависит от профессии)                    |
| LUNCH      | 12–13   | Обедает на кухне                                         |
| WORKING    | 13–18   | Вторая рабочая смена                                     |
| FREE_TIME  | 18–22   | Свободное время: курилка, кухня, зал, бродит             |
| TRAVELING  | всегда  | Путники/паломники/охотники бродят по лабиринту           |
| HIDING     | самосбор| Прячется в жилой комнате за гермодверями                 |

#### Рабочие комнаты по профессиям

| Занятие        | Рабочее место                 | Доля  |
|----------------|-------------------------------|-------|
| Домохозяйка    | Жилая, кухня                  | 10%   |
| Слесарь        | Цех                           | 10%   |
| Секретарь      | Бухгалтерия                   | 10%   |
| Электрик       | Цех                           | 10%   |
| Повар          | Кухня                         | 5%    |
| Врач           | Медпункт                      | 5%    |
| Токарь         | Цех                           | 10%   |
| Механик        | Цех                           | 10%   |
| Кладовщик      | Кладовая                      | 10%   |
| Алкоголик      | Курилка, зал, кухня           | 5%    |
| Учёный         | Бухгалтерия, медпункт         | 5%    |
| Ребёнок        | Жилая, зал                    | 10%   |
| Директор       | Бухгалтерия, зал              | 1%    |
| Путник         | — (бродит по лабиринту)       | —     |
| Паломник       | — (бродит по лабиринту)       | —     |
| Охотник        | Коридоры, залы                | —     |

Каждая профессия имеет **уникальный процедурный спрайт** (каска, халат, фартук, галстук, очки и т.д.). Ребёнок рендерится в 60% размере (spriteScale 0.6) и двигается медленнее.

#### Переходы между состояниями

- Автоматические по расписанию (час → состояние)
- Прерывания: срочные потребности (туалет, голод) прерывают работу
- Ранение: NPC с <50% ХП идёт в медпункт вместо работы
- Самосбор: граждане и учёные прячутся (HIDING), **ликвидаторы и культисты НЕ прячутся** — продолжают действовать
- Путники/паломники/охотники всегда в состоянии TRAVELING (кроме граждан-путников, которые прячутся при самосборе)

#### Странники (Travelers)

Помимо оседлых NPC, по лабиринту бродят автономные странники — **не привязаны к квартирам**, спавнятся в коридорах:

| Тип       | Фракция      | Кол-во | Поведение при самосборе                     |
|-----------|--------------|--------|---------------------------------------------|
| Путник    | Граждане     | 32     | Прячется в ближайшей жилой комнате          |
| Паломник  | Культисты    | 24     | **Не прячется** — продолжает бродить        |
| Охотник   | Ликвидаторы  | 24     | **Не прячется** — продолжает бродить        |

Странники перемещаются между случайными комнатами мира, удовлетворяя потребности по пути (кухня, санузел, медпункт).

#### NPC потребности (упрощённая система)

NPC восстанавливают потребности **просто находясь в нужной комнате**:

| Потребность | Комната восстановления    |
|-------------|---------------------------|
| Еда         | Кухня                     |
| Жажда       | Кухня, санузел            |
| Сон         | Жилая (только ночью)      |
| Туалет      | Санузел                   |
| Здоровье    | Медпункт                  |

### Потребности игрока (Needs)

Игрок управляет потребностями вручную:
- **Еда** 0–100 — падает со временем, при 0 теряется здоровье
- **Вода** 0–100 — падает быстрее еды
- **Сон** 0–100 — при <10 скорость ×0.5
- **Туалет** (pee/poo) 0–100 — растёт со временем

### Предметы

- **Еда:** хлеб, тушёнка, каша, сырое мясо
- **Напитки:** вода, чай, компот
- **Медицина:** бинт, таблетки, антидепрессант
- **Ближний бой:** кулаки (3), нож (8), ключ гаечный (12), труба (18), арматура (25), топор (30)
- **Огнестрельное:** Макаров (20, 9мм), обрез (8×6, дробь), гвоздомёт (12, гвозди)
- **Патроны:** 9мм, дробь, гвозди
- **Сгустки (ПСИ):** пси-руны, экипируются как оружие, используют ПСИ вместо патронов
- **Сюжетные:** Идол Чернобога (128 шт. в мире)
- **Прочее:** фонарик, туалетная бумага, сигареты, книга, записки, ключи

### Сгустки — ПСИ-система

**Сгустки** — кристаллизованная пси-энергия хруща, материализующаяся в лабиринте. Экипируются как оружие (клавиша E в инвентаре), при атаке (Пробел/ЛКМ) потребляют **ПСИ** вместо патронов. Находятся в медпунктах, бухгалтериях, залах или выдаются за квесты. **ПСИ не регенерируется само** — восстанавливается только через предметы (антидепрессант +20 ПСИ, таблетки +5 ПСИ). Максимум ПСИ зависит от уровня + INT.

| Сгусток              | ПСИ | Тип       | Эффект                                                    |
|----------------------|-----|-----------|-----------------------------------------------------------|
| Пси удар             | 1   | Снаряд    | Фиолетовый болт, 10 урона. Базовая пси-атака              |
| Разрыв связности     | 3   | АоЕ снаряд| Фиолетовый болт, 10 урона + взрыв радиус 3                |
| Пси буря             | 10  | Мгновенно | 10 урона всем в поле зрения (конус ~60°, 12 клеток)       |
| Выжиг мозга          | 8   | Мгновенно | Мгновенная смерть цели ≤ вашего уровня                    |
| Безумие              | 5   | Мгновенно | Цель нападает на всех 60с (1 игровой час)                  |
| Контроль             | 8   | Мгновенно | Цель становится союзником 60с                              |
| Фазовый сдвиг        | 8   | Мгновенно | Проходить сквозь стены 60с                                 |
| Метка                | 3   | Мгновенно | Запомнить позицию для телепорта                             |
| Возврат              | 3   | Мгновенно | Телепорт к сохранённой метке                               |

**Баланс:** на 1 уровне ПСИ=10 — можно 10 раз ударить Пси ударом или 1 раз применить Пси бурю. INT увеличивает максимум ПСИ на 10% за очко. **ПСИ не регенерируется само** — используйте Антидепрессант (+20 ПСИ) и Таблетки (+5 ПСИ).

**Лор:** сгустки — побочный продукт самосбора. Стены хруща концентрируют психическую энергию пространства в осязаемые сгустки. Учёные и врачи коллекционируют их. Культисты поклоняются им.

#### Матка — босс-спавнер

**Матка** — особый монстр, который появляется как **босс фиолетового тумана** при самосборе с вероятностью **10%** (вместо обычных боссов: Бетонник/Арматура/Кошмарище). Выглядит как огромная ползущая женщина с раздутым телом, фиолетовыми венами и пустыми глазами.

- **Каждые 60 реальных секунд** (1 игровой час) рожает случайного монстра рядом с собой
- Порождённые монстры: Сборка, Тварь, Мертвяк, Теневик или Ползун
- Порождённые монстры масштабируются по уровню зоны (RPG stats)
- При рождении выводится сообщение: «Матка родила {имя монстра}!»
- Убийство Матки-босса тумана **очищает фиолетовый туман** в захваченной зоне

### RPG-система (уровни, атрибуты, масштабирование)

Все NPC и монстры имеют **RPG-статистику**: уровень, XP, атрибуты. Масштабирование привязано к **уровню зоны** — чем глубже в лабиринт, тем сильнее враги.

#### Формула XP

Мягкая квадратичная: `xpForLevel(L) = 80×L + 5×L²`

| Уровень | XP до следующего |
|---------|------------------|
| 1       | 85               |
| 5       | 525              |
| 10      | 1 300            |
| 20      | 3 600            |
| 50      | 16 500           |

#### Атрибуты (STR, AGI, INT)

Каждый уровень даёт **+1 очко атрибута** (клавиши `1/2/3` в инвентаре):

| Атрибут        | Эффект (за очко)                                   |
|----------------|-----------------------------------------------------|
| **Сила (STR)** | +10% урон ближнего боя, +10% макс. ХП              |
| **Ловкость (AGI)** | +10% скорость движения, +10% скорость атаки    |
| **Интеллект (INT)** | +10% бонус XP, +10% макс. PSI                 |

#### Базовые характеристики

- **ХП:** 100 base + 10 за уровень, ×(1 + 0.1 × STR)
- **PSI:** 10 base + 1 за уровень, ×(1 + 0.1 × INT)
- **XP за монстра:** `base × (1 + 0.15 × (уровень_монстра - 1))`

#### Масштабирование монстров

Монстры, спавнящиеся в зонах, получают **RPG-статистику** по уровню зоны:
- ХП масштабируется: `baseHp × (1 + 0.15 × (level - 1))`
- Урон масштабируется: `baseDmg × (1 + 0.1 × (level - 1))`
- Скорость масштабируется: `baseSpeed × (1 + 0.05 × (level - 1))`

### САМОСБОР

Интервал зависит от этажа: **Жилая** 5–10 мин, **Коллекторы** 3–7 мин, **Ад** 1–5 мин. Длительность **случайная: 12–90 реальных секунд**:

1. **Граждане и учёные переходят в HIDING** — бегут в жилые комнаты. **Ликвидаторы и культисты НЕ прячутся.**
2. **Двери герметично запираются**
3. **Захват зоны** — случайная не-SAMOSBOR зона захватывается фиолетовым туманом навсегда
4. **Босс тумана** — в захваченной зоне спавнится босс: **90%** шанс на Бетонника/Арматуру/Кошмарище, **10%** на Матку. Убийство босса **очищает туман** в зоне
5. **Фиолетовый туман** засевается в центре зоны (радиус ~5), расширяется BFS каждые 0.15с
6. **Монстры спавнятся в тумане** (fog > 100) каждые 2с — количество растёт с каждым самосбором
7. Из стен вылезают **коридорные монстры** (6+ за самосбор, эскалация по типам)
8. После окончания → двери размыкаются
9. **Лабиринт перестраивается** — volatile часть уничтожается и генерируется заново
10. **Монстры сохраняются** — при перестройке удаляются только снаряды, монстры остаются
11. **Квестодатели перевыбираются** — 10% NPC получают новые квесты
12. NPC возвращаются к расписанию

#### Эскалация монстров по самосборам

| Самосбор | Доступные монстры                                        |
|----------|----------------------------------------------------------|
| 1+       | Сборка, Тварь, Ползун                                    |
| 2+       | + Мертвяк, Теневик                                      |
| 3+       | + Бетонник, Глаз, Кошмарище                              |
| 5+       | + Арматура                                               |

## Правила дизайна

### Инкапсуляция комнат стенами (КРИТИЧНО)

**Все функциональные комнаты обязаны быть полностью окружены стенами и однокле́точными дверями** (паттерн стена-дверь-стена). Никаких дыр в стенном периметре. Это жизненно важно: при самосборе NPC прячутся за гермодверями, дыра в стене = смерть жителей. После генерации `repairRoomWalls()` проверяет и заделывает любые пробоины.

### Фракции

| Фракция       | Описание                                                |
|---------------|---------------------------------------------------------|
| Граждане      | Обычные жители. Едят, спят, работают. Нейтральные       |
| Ликвидаторы   | Борются с монстрами и последствиями самосборов          |
| Дикие         | Анархисты и бандиты. Враждебны ко всем                  |
| Культисты     | Поклоняются самосбору. ~30-40% вооружены Сгустками (ПСИ). Агрессивны к ликвидаторам |
| Учёные        | Изучают хрущ. Дают квесты, торгуют информацией          |

### Фракционная война (S.T.A.L.K.E.R.-стиль)

Фракции **контролируют территорию** и ведут боевые действия друг с другом. Каждая зона имеет фракцию-владельца, а NPC патрулируют и защищают свои зоны.

#### Штабы (HQ)

Каждая фракция (кроме учёных) получает **штабную комнату** в одной из своих зон. Штаб — укреплённая комната с уникальной текстурой стен, лампами и гарнизоном.

#### Патрули

Фракции спавнят **патрульные отряды** (2–4 бойца) в своих зонах. Патрульные перемещаются между комнатами, охраняя территорию. При встрече с враждебной фракцией вступают в бой.

#### Боевая система NPC

- NPC сканируют ближайших врагов (радиус ~12 клеток) с кэшированием цели (`combatTargetId`) и кулдауном пересканирования (`combatScanCd`)
- Враждебность определяется отношениями между фракциями (матрица `relations`)
- Урон по NPC вызывает **штраф отношений** между фракциями атакующего и жертвы
- Убийства логируются в HUD

#### Захват зон

При самосборе фиолетовый туман **навсегда захватывает** случайную зону. Фракция зоны становится `SAMOSBOR`. Территория постепенно сжимается.

### Именование NPC (фракционная система)

Имена NPC генерируются **в зависимости от фракции** с учётом пола:

| Фракция      | Стиль                           | Примеры                                    |
|--------------|----------------------------------|--------------------------------------------|
| Граждане     | Русские имена (имя + фамилия)   | Дмитрий Козлов, Ольга Петрова              |
| Ликвидаторы  | Военные звания + фамилия         | Рядовой Сидоров, Сержант Кузнецова         |
| Дикие        | Имя + прозвище в кавычках        | Дима «Бетон», Машка «Шаболда»             |
| Культисты    | Элдритч-титулы (прил. + сущ.)    | Чёрная Гниль, Кровавый Идол               |
| Монстры      | Процедурная слоговая генерация   | Хрызгот, Блурча, Шкварглок                |

- Пол персонажа (`isFemale`) определяется при генерации имени
- Сообщения об убийствах учитывают род: «убил» / «убила», «повержен» / «повержена»

### Отношения (Relations)

- **NxN матрица** Int8Array — отношение каждого к каждому: -128..127 (включая игрока id=0)
- Стартовые: семья +60, фракция +20, межфракционные бонусы/штрафы
- Динамика: помощь → рост, агрессия → падение, квесты → ±отношения

### Торговля (Trade)

Бартерная система через **меню NPC** (вкладка «Торговля»):
- Товары генерируются по профессии (повар — еда, врач — медикаменты и т.д.), 2–5 предметов
- **Покупка** — обмен случайного предмета игрока на выбранный; **продажа** — передача предмета NPC

### Сохранение / Загрузка

- **Enter → меню**: продолжить, новая игра, сохранить, загрузить
- `localStorage` (`gigahrush_save`): позиция, ХП, потребности, инвентарь, время, квесты, этаж
- При загрузке мир перегенерируется для сохранённого этажа

### Квесты

Процедурные квесты от NPC. **Только ~10% NPC могут давать квесты** — остальные отвечают «Мне нечего тебе поручить». После каждого **самосбора** квестодатели перевыбираются заново (re-roll 10%). Макс **5 активных квестов** одновременно. Требуется отношение ≥ -10.

| Тип   | Вес  | Описание                              | Отношения |
|-------|------|---------------------------------------|-----------|
| FETCH | 40%  | «Принеси мне [предмет]»              | +10       |
| VISIT | 25%  | «Сходи в [комнату #N]»               | +8        |
| KILL  | 20%  | «Убей [монстра] в зоне»              | +15       |
| TALK  | 15%  | «Передай [NPC] сообщение»            | +12       |

Квесты зависят от занятия NPC: повар просит еду, врач — медикаменты, охотник — убить монстра.

### Экономика (Рубли)

- Каждый NPC и игрок имеют **деньги (₽ рубли)**
- Игрок стартует с **100₽**
- NPC получают стартовые деньги по профессии: директор 200–500₽, ребёнок 0–10₽, остальные 20–100₽
- Каждый предмет имеет **цену (value)** — отображается в инвентаре
- Рубли отображаются в HUD (нижняя панель + инвентарь)

### Звук

Процедурный (Web Audio API, без файлов):
- Шаги, удар, скрип двери, рык монстра, сирена самосбора, звук подбора, фоновый дрон

### Оптимизации

- **Pre-allocated BFS**: `Uint16Array` + generation counter для visited, `Int32Array` для prev — нулевые аллокации при навигации
- **Кэширование боевых целей**: `combatTargetId` + `combatScanCd` на AIState — NPC не пересканируют врагов каждый кадр
- **dist2()**: квадрат расстояния вместо `Math.sqrt` для всех проверок дальности
- **entityById Map**: пересобирается раз в кадр, O(1) поиск по id вместо O(n) scan
- **Зональный пересчёт**: `recalcZoneControl` обновляет только затронутые зоны

## Гайд для дизайнеров: как добавлять контент

Архитектура модульная — новый контент добавляется **без изменения существующих файлов**. Каждый контент-модуль — автономный `.ts` файл, который подключается одной строкой в оркестраторе.

### Добавить новую комнату на жилом этаже

1. Создай файл `src/gen/living/my_room.ts`:

```typescript
import { Cell, Tex, RoomType, Feature, type Room, type Entity, EntityType, AIGoal, Faction, Occupation } from '../../core/types';
import { World } from '../../core/world';
import { freshNeeds } from '../../data/catalog';
import { stampRoom } from '../shared';

export function generateMyRoom(
  world: World, nextRoomId: number, entities: Entity[], nextId: { v: number },
): { room: Room; nextRoomId: number } {
  // 1. Штампуем комнату (rx, ry — координаты, w×h — размер)
  const room = stampRoom(world, nextRoomId++, RoomType.COMMON, 500, 500, 10, 8, -1);
  room.name = 'Моя комната';
  room.wallTex = Tex.PANEL;
  room.floorTex = Tex.F_CARPET;

  // 2. Защищаем aptMask (если не должна разрушаться при самосборе)
  for (let dy = -1; dy <= 8; dy++)
    for (let dx = -1; dx <= 10; dx++)
      world.aptMask[world.idx(500 + dx, 500 + dy)] = 1;

  // 3. Стены / текстуры / фичи
  world.features[world.idx(505, 504)] = Feature.LAMP;

  // 4. NPC (опционально)
  entities.push({
    id: nextId.v++, type: EntityType.NPC,
    x: 505.5, y: 503.5, angle: 0, pitch: 0, alive: true, speed: 1.2,
    sprite: Occupation.COOK, name: 'Повар Иванов',
    needs: freshNeeds(), hp: 100, maxHp: 100, money: 30,
    ai: { goal: AIGoal.IDLE, tx: 0, ty: 0, path: [], pi: 0, stuck: 0, timer: 0 },
    inventory: [{ defId: 'bread', count: 3 }],
    faction: Faction.CITIZEN, occupation: Occupation.COOK,
    canGiveQuest: true, questId: -1,
  });

  // 5. Предметы (опционально)
  entities.push({
    id: nextId.v++, type: EntityType.ITEM_DROP,
    x: 503.5, y: 503.5, angle: 0, pitch: 0, alive: true, speed: 0,
    sprite: 16, spriteScale: 1.0,
    inventory: [{ defId: 'canned', count: 2 }],
  });

  return { room, nextRoomId };
}
```

### Принципы

- **Один файл = один контент-модуль.** Всё своё: планировка, текстуры, NPC, предметы, квесты.
- **Каждый сюжетный NPC — отдельный `.ts` файл.** Пример: `start_room.ts` (Ольга + Барни), `yakov_lab.ts` (Яков Давидович). **Не мешать сюжетных NPC в `npcs.ts`** — `npcs.ts` занимается только массовым спавном (предметы, семьи, путники).
- **Никогда не меняй `shared.ts`** — это общие утилиты. Используй `stampRoom`, `placeDoor`, `carveCorridor`.
- **`aptMask = 1`** — комната не разрушается при самосборе. Без маски — перестроится вместе с лабиринтом.
- **Подключение: одна строка в `index.ts`** оркестраторе этажа. Сигнатура: `(world, nextRoomId, entities, nextId, ...)` → `{ room, nextRoomId }`.
