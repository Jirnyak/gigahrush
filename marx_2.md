#### 1. Определение события в слое Core / Events
Необходимо расширить или убедиться в наличии типа события в системе эвентов, чтобы НПЦ могли реагировать на действие игрока без жесткой связки. Это позволит легко расширять функционал в будущем (например, реакция фракций).

```typescript
// Заглушка для src/systems/events.ts или src/core/types.ts
export interface EventUrinatePublic {

    y: number;
    roomId: RoomId | null; // В какой комнате это произошло
    isBathroom: boolean;   // Находился ли игрок в туалете

```

#### 2. Генерация события (Слой Input / Interactions)
При нажатии клавиши `P` (уже есть функционал спавна лужи), необходимо задиспатчить событие. Если текущий тайл принадлежит чужой зоне и это не туалет — это публичное нарушение порядка, которое может увидеть любой НПЦ поблизости.

```typescript
// Заглушка для src/systems/interactions.ts (обработчик 'P')
import { publishEvent } from './events';
import { getRoomAt } from '../core/world_utils';

export function handlePlayerUrinate(player: Player, world: World) {
    const room = getRoomAt(world, player.x, player.y);
    const isBathroom = room?.type === RoomType.BATHROOM;
    
    // Существующий код: Спавн следа/лужи
    // spawnPuddle(world, player.x, player.y);
    
    // ПУБЛИКУЕМ СОБЫТИЕ ДЛЯ ИИ
    publishEvent(world, {
        type: 'urinate_public',
        sourceId: player.id,
        x: player.x,
        y: player.y,
        roomId: room?.id ?? null,
        isBathroom
    });
    
    // Если это публичное место (не туалет) — снижаем глобальную карму
    if (!isBathroom) {
        player.karma -= 2; // Значение может балансироваться через src/data/balance.ts
    }
}
```

#### 3. Реакция ИИ (Слой Systems / AI)
В модуле обработки A-Life или утилити ИИ НПЦ (`src/systems/ai/npc_utility.ts` или `src/systems/ai/ai_events.ts`), нужно добавить подписку на это событие, чтобы каждый НПЦ в радиусе видимости отреагировал должным образом.

```typescript
// Заглушка для src/systems/ai/npc_events_reaction.ts
import { World, Npc, AiState, Faction } from '../../core/types';

export function processPublicUrinateEvent(world: World, npc: Npc, event: EventUrinatePublic) {
    if (event.isBathroom) return; // В туалете можно, реакций нет
    
    // Проверка дистанции (Iron Law: используем dist2 для скорости)
    const dx = npc.x - event.x;
    const dy = npc.y - event.y;
    const dist2 = dx * dx + dy * dy;
    
    const REACTION_RADIUS = 8;
    
    if (dist2 <= REACTION_RADIUS * REACTION_RADIUS) {
        // У НПЦ должно быть поле playerRelation
        if (npc.faction !== Faction.WILD) {
            npc.playerRelation = (npc.playerRelation || 0) - 15;
            
            // Если отношение упало ниже порога — переводим в агрессию
            const HOSTILE_THRESHOLD = -30;
            if (npc.playerRelation <= HOSTILE_THRESHOLD) {
                npc.aiState = AiState.ATTACKING;
                npc.targetId = event.sourceId;
                
                // Опционально: вызов фразы (barks) через систему локализации
                // publishEvent(world, {
                //     type: 'npc_bark',
                //     npcId: npc.id,
                //     textId: 'bark_angry_urinate'
                // });
            }
        }
    }
}
```

### 📋 Критерии приемки (DoD - Definition of Done)
Для того чтобы задача считалась выполненной (чтобы Оркестратор одобрил ваш PR), убедитесь, что соблюдены следующие условия:
1. **Событие генерируется**: При нажатии клавиши P (или другой, назначенной на действие) создается и публикуется в шину `urinate_public`.
2. **Радиус реакции работает**: НПЦ, находящиеся в пределах 8 тайлов (вычисляется через `dist2`), получают уведомление и обрабатывают его.
3. **Учет туалетов**: Если комната, в которой находится игрок, имеет тип `RoomType.BATHROOM` (или аналогичный по смыслу), НПЦ **игнорируют** событие и **не** понижают отношение.
4. **Враждебность**: Если отношение НПЦ (`playerRelation`) опускается ниже определенного порога (например, `-30`), он начинает атаковать игрока (переключает свой AI State на `ATTACKING`).
5. **Тестируемость**: Вы не сломали существующие тесты. Опционально: в игру добавлен юнит-тест (в директории `tests/`), который проверяет логику понижения отношения НПЦ при событии `urinate_public` в зависимости от расстояния.
