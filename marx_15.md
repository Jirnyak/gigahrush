## Текущая задача
**Арена: НПЦ против НПЦ. Честная генерация бойцов со случайным лутом и симуляция их битвы.**

### 1. Глубокий контекст задачи и бизнес-логика
Вам необходимо реализовать функционал гладиаторских боев на арене (построенной в задаче marx_13). Суть фичи: игрок подходит к NPC-организатору, запускает бой. Система должна честно сгенерировать двух НПЦ-бойцов (гладиаторов), выдать им процедурное снаряжение (оружие, броню) случайного качества, поместить их на ринг и запустить симуляцию битвы в реальном времени.

Никакой скриптованной абстракции или предрасчетов: используется та же самая боевая AI-система, что и в обычной игре. Камера переключается в "свободный режим" и центрируется на ринге, чтобы игрок мог наблюдать за битвой. Победитель выживает, труп проигравшего остается (с лутом, который игрок сможет собрать, если спрыгнет на ринг, нарушив правила), система публикует событие о завершении боя.

```typescript
export function spawnArenaGladiator(world: World, roomId: string): Entity {
    // 1. Генерируем случайное снаряжение (Mid tier)
    const loadout = generateProceduralLoadout(world, LoadoutTier.MID);
    
    // 2. Создаем Entity
    const gladiator = new Entity({
        type: EntityType.NPC,
        faction: Faction.GLADIATOR, // Спец-фракция для арены
        name: generateRandomName(),
        inventory: loadout.items,
        weapon: loadout.weapon,
        hp: 120 + Math.random() * 50 // Случайный разброс ХП
    });
    
    // 3. Спавним на арене (в противоположных углах)
    placeEntityInRoomCorner(world, gladiator, roomId);
    return gladiator;
}
```

### 3. Пошаговый план внедрения (Action Plan)
1. **Генерация Гладиаторов:** Реализуйте функцию `spawnArenaGladiator`, которая использует таблицы лута для одевания бойцов. Убедитесь, что они получают патроны для огнестрела.
2. **ИИ и Аггро:** Обновите боевой ИИ (если нужно) так, чтобы он всегда ставил `combatTargetId` в абсолютный приоритет над фракционными отношениями.
3. **Логика Дуэли:** Создайте контроллер дуэли в `arena_combat.ts`, который спавнит двоих, задает им таргеты и публикует события старта и завершения.
4. **Управление камерой:** Используйте методы `setFreeCamera` / `restorePlayerCamera`. Во время `setFreeCamera` управление игроком должно быть заблокировано (или он должен просто стоять на трибунах).
5. **Очистка и Награды:** После смерти одного из НПЦ, победитель должен восстановить немного ХП (опционально) и через некоторое время исчезнуть (вернуться в пул / деспавн), оставив труп проигравшего на арене.

1. Выполните проверку типов: `npm run typecheck`.

   - Бойцы должны начать стрелять/бить друг друга с использованием процедурного оружия.
   - Дождитесь смерти одного из бойцов.
   - Камера должна вернуться к игроку. Убедитесь, что игра не крашится из-за отсутствия таргета у выжившего.

### 5. Возможные Edge Cases и Подводные Камни
- **LOS (Line of Sight):** Убедитесь, что геометрия ринга не блокирует видимость бойцов друг для друга, иначе они будут просто стоять и тупить.
- **Бесконечный бой:** Если сгенерировались два бойца с высокой броней и слабым melee-оружием, бой может идти 10 минут. Добавьте тайм-аут дуэли или искусственное увеличение урона со временем ("Усталость").
- **Вмешательство игрока:** Если игрок начнет стрелять с трибун по гладиаторам, ИИ гладиаторов может переагриться. Сделайте гладиаторов неуязвимыми для внешнего урона, либо штрафуйте игрока за нарушение правил (сразу агрессия всех ликвидаторов).
