### Шаг 2. Разработка ИИ Шахтера (Gnome Miner)
1. В `src/systems/ai.ts` добавьте ветку поведения для `AIType.GNOME_MINER`.
2. Логика:
   - Гном периодически ищет соседний блок стены (по определенным критериям: не гермодверь, не граница карты).
   - "Бьет" стену (проигрывает анимацию/звук).
   - При разрушении стены (смена в `world.walls`) гном может заспавнить кусок руды.
   - Альтернативно: гном ставит блок в пустом пространстве, строя лабиринт.

```typescript
// Stub: src/systems/ai_gnome.ts
export function processGnomeAi(gnome: Entity, world: World): void {
    if (gnome.cooldown > 0) return;
    
    // Find adjacent drillable wall
    const targetCell = findDrillableWallNear(world, gnome.idx);
    if (targetCell !== -1) {
        drillWall(world, targetCell);
        gnome.cooldown = 60; // Wait before next drill
    } else {
        // Wander to find new walls
        wander(gnome, world);
    }
}

function drillWall(world: World, idx: number) {
    world.walls[idx] = 0;
    // CRITICAL: bump versions to trigger rebuilds
    world.surfaceVersion++;
    invalidateNavCache(world, idx);
    publishEvent('wall_mined', { idx });
}
```

### Шаг 3. Защита от разрушения критической инфраструктуры
1. Гномы не должны разрушать маску `protected` (сюжетные комнаты, лифты).
2. Функция `findDrillableWallNear` должна строго проверять `world.aptMask[idx]` или аналогичные флаги неразрушимости.
