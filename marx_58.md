## Текущая задача
**Рендер: Рои Мух (3D облака над трупами).**

### Контекст задачи
Финальная часть поведения critters: Мухи (`fly`). В отличие от крыс и тараканов, мухи не ползают по полу, а образуют трехмерные рои (орбиты) над мертвыми телами (трупами) и органическим мусором. Это добавляет гнетущей атмосферы. Рисуются как мельтешащие чёрные точки. Сопровождаются жужжанием в радиусе 5 клеток.

### Детальная спецификация по Архитектуре и Реализации

#### 1. Модификация `src/render/critters.ts`
Добавьте поддержку мух и их 3D-орбитального движения.
Мухам нужен "якорь" (anchorX, anchorY) — координаты трупа.
```typescript
// Stub for Critter update loop
export function updateFlies(dt: number) {
    const time = performance.now() / 1000;
    
    for (let i = 0; i < CRITTERS_POOL.length; i++) {
        const c = CRITTERS_POOL[i];
        if (!c.active || c.type !== 'fly') continue;
        
        // c.targetX, c.targetY store the anchor (corpse location)
        // c.phase - random phase shift
        // c.speed - orbit speed multiplier
        
        const r = 0.2 + (Math.sin(time * 2 + c.phase) * 0.1); // Radius pulsing
        c.x = c.targetX + r * Math.cos(time * c.speed + c.phase);
        c.y = c.targetY + r * Math.sin(time * c.speed + c.phase);
        c.z = 0.4 + 0.2 * Math.sin(time * 3 + c.phase); // Height modulation (0.2 - 0.6)
    }
}
```

#### 2. Жужжание (Audio Logic)
Жужжание должно вычисляться на лету без создания 30 аудио сурсов.
Достаточно в конце `updateFlies` найти ближайшего "якоря" мух к игроку:
```typescript
// Inside render loop or audio system
let minFlyDist = Infinity;
// find min dist to player among active flies...

if (minFlyDist < 5.0) {
    const volume = 1.0 - (minFlyDist / 5.0); // fade out at 5 cells
    publishEvent({ type: 'audio_loop', soundId: 'fly_buzz', volume: volume, id: 'ambient_buzz' });
} else {
    publishEvent({ type: 'audio_stop', id: 'ambient_buzz' });
}
```

#### 3. Визуализация в `src/render/webgl.ts`
В `drawCritters` мухи должны рисоваться как черные (или темно-зеленые) точки размером 1x1 или 2x2 px.
Обязательно учитывайте их `c.z` для проекции (высота над полом).

#### 4. Лимиты
- Максимум 30 мух на экран, максимум 10 мух над одним трупом.
- Мухи появляются не сразу после смерти, а через 30 секунд (реализуется через таймер спавна в marx_59, здесь только логика движения).

#### 5. Тестирование и валидация
- Убедитесь, что мухи не вылетают за пределы радиуса 0.5 от якоря (не "улетают в закат" из-за ошибки в sin/cos).
- Запустите `npm run typecheck` и `npm run check:full`.
- Проверьте отсутствие проблем с производительностью: sin/cos 30 раз в кадр не является бутылочным горлышком, но избегайте вложенных циклов.
