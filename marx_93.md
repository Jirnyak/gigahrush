## Текущая задача
**Отказ от эгоцентризма игрока**

### Контекст задачи
На данный момент скрипты и события часто триггерятся только когда игрок подходит близко. Это создает ощущение "театральных декораций", которые ждут главного героя. Необходимо, чтобы мир жил сам по себе. Стычки мутантов и сталкеров, торговля НПЦ, смерти и лутинг должны происходить глобально на этаже независимо от присутствия игрока. Игрок может прийти на место битвы и найти только трупы и гильзы.

### Конкретные файлы и паттерны
- **`src/systems/events.ts`** (глобальная шина событий)
- **`src/systems/alife.ts`** (жизнь НПЦ)
- **`src/systems/ai_combat.ts`** (оффскрин-бои)

## Подробный Roadmap (Jules)
Этот план описывает шаги для отвязки игрового мира от присутствия игрока.

### 1. Переработка триггеров
- Проверьте существующие `ZoneTrigger` и квестовые маркеры. Убедитесь, что они не блокируют обычное течение жизни (например, НПЦ не стоят вечно в ожидании, пока игрок с ними заговорит, если это не строго сюжетный персонаж).
- Введите концепцию "Global Timeline" — события развиваются по времени, а не по дистанции до игрока.

### 2. Заглушки TypeScript (TypeScript Stubs)
Создайте абстракцию для событий, происходящих вне поля зрения (Off-screen events):

```typescript
// src/systems/events.ts
export enum OffscreenEventType {
    FACTION_SKIRMISH = 'FACTION_SKIRMISH',
    STALKER_DEATH = 'STALKER_DEATH',
    MUTANT_NEST_CLEARED = 'MUTANT_NEST_CLEARED'
}

export interface OffscreenEvent {
    id: string;
    type: OffscreenEventType;
    x: number;
    y: number;
    participants: number[]; // ID сущностей
    timeOccurred: number;
    resolved: boolean;
}
```

### 3. Легковесная симуляция боев вне экрана
- Если две враждебные группы сущностей сталкиваются далеко от игрока, полная симуляция пулек и коллизий (raycasting) отключена ради оптимизации.
- Вместо этого используйте "автобой" (Auto-resolve) на основе статов.
- Напишите `resolveOffscreenCombat(groupA, groupB)`:

```typescript
// src/systems/ai_combat.ts
export function resolveOffscreenCombat(entities: Entity[]): void {
    // 1. Оценить боевую мощь (оружие, броня, здоровье).
    // 2. Бросить виртуальные кубики.
    // 3. Убить проигравших, уменьшить здоровье победителям, заспавнить лут на клетках.
}
```

### 4. Информационный след
- Игрок должен узнавать о событиях, которые он пропустил.
- При смерти НПЦ отправляйте сообщение в PDA/Инфосеть (если НПЦ был известен).
- В месте битвы генерируйте кровь, гильзы (decals) и трупы, которые со временем превращаются в скелеты или съедаются.

### 5. Независимые расписания (Schedules)
- Дайте торговцам и лидерам фракций реальное расписание. Если игрок приходит в магазин ночью, торговец спит, дверь заперта. Игрок должен подстраиваться под мир, а не мир под игрока.
- Обновите `src/systems/alife.ts` для поддержки поля `schedule` в `ALifeState`.

### 6. Оптимизация
- Оффскрин симуляция не должна выполняться каждый кадр. Выполняйте `resolveOffscreenCombat` пакетами (batch) раз в несколько секунд (или тиков сервера).
- Убедитесь, что отрисовщик (`render/`) не пытается рисовать оффскрин события (отсечение по фрустуму уже должно работать, но проверьте).

### 7. Тестирование
- Напишите тест, симулирующий этаж без игрока на 1000 тиков. Проверьте, что количество живых НПЦ изменяется из-за внутренних стычек.
- Запустите `npm run check`.

### 9. Инструкция по написанию Unit-тестов (Обязательно)
Чтобы гарантировать стабильность архитектуры ГИГАХРУЩА, вы (агент Jules) обязаны:
1. Создать файл тестирования в директории `tests/` (если он еще не существует).
2. Написать моки для зависимых систем (например, мок глобального времени или мок генератора случайных чисел).
3. Покрыть тестами краевые случаи (edge cases):
   - Что произойдет, если сущность будет уничтожена прямо во время обновления?
   - Как система ведет себя на границах этажа (координаты < 0 или >= 1024)?
4. Запустить `npm run test:unit` и убедиться, что ваш код не сломал существующие тесты.

### 11. Заключительная валидация на мобильном устройстве
Так как ГИГАХРУЩ имеет браузерный таргет и ориентирован на мобильные устройства:
- Проверьте, не вызывает ли ваша фича утечек памяти (запустите `npm run check:browser`).
- Массивы, которые итерируются каждый кадр, должны использовать typed arrays (`Int32Array`, `Float32Array`), если в них хранятся только числа (ID, дистанции, флаги состояний).
- Убедитесь, что все новые UI-элементы поддерживают touch-события и корректно масштабируются на узких экранах.
