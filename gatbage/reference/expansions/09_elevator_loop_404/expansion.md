# Expansion 09: Лифтовая петля 404

Версия: 0.1 design  
Статус: технический дизайн для разработки  
Роль в линейке: номерные этажи, ошибки маршрута, память и procedural pocket content  
Базовые источники: `README.md`, `desdoc.md` разделы 404, 556, 777, 1337, лифты, floorInstance, слухи

## Назначение

«Лифтовая петля 404» превращает городские легенды о номерных этажах в управляемую систему карманных уровней. Игрок не получает «1000 этажей» как enum и не гуляет по бесконечной таблице мемов. Он сталкивается с ошибкой маршрута: лифт открывается там, где этажа быть не должно, карта показывает пустоту, NPC помнят разные номера, а выход требует понять правило конкретной петли.

Экспаншн нужен для поздней игры и для связки метро, архивов, слухов и Пустоты. Он должен дать ощущение, что гигахрущ глубже реализации, но при этом иметь строгий технический MVP.

## Игровая задача

Текущая игра уже имеет несколько `FloorLevel` и лифты. Расширение добавляет временные `floorInstance`, которые появляются из ошибки маршрута, слуха, документа или самосборного сбоя. Это позволяет реализовывать 404, 556, 777 и 1337 как компактные procedural pockets без раздувания базового floor enum.

Игрок учится относиться к лифту как к системе риска. Нажатие кнопки больше не всегда означает переход на известный этаж.

## Основная игровая петля

Игрок собирает зацепку: карта без номера, билет метро, архивный приказ, слух радиокружка или запись о несуществующем этаже. Затем он провоцирует ошибку лифта: выбирает неправильную кнопку, едет во время самосбора, использует испорченный пропуск или следует проводнику. Лифт открывает pocket floor. Внутри нужно понять локальное правило, выполнить короткую цель и найти выход.

Правило всегда простое, но странное: на 404 нельзя доверять карте; на 556 надо читать старые протоколы; на 777 опасна чрезмерная безопасность; на 1337 доступ меняется через радиокоды. Игрок выходит с ресурсом, документом, слухом или травмой памяти.

## Номерные этажи MVP

MVP включает четыре петли:

| Петля | Правило | Награда | Риск |
| --- | --- | --- | --- |
| 404 «Не найден» | карта и маркеры лгут, выход через неправильную дверь | потерянные предметы, путь к архивной ошибке | заблудиться и выйти в опасную зону |
| 556 «П-46» | старые инструкции работают лучше новых | протокол самосбора, слухи о каноне | высокая частота классического тумана |
| 777 «Счастливый» | безопасность является ловушкой | отдых/еда/редкая передышка | долг или memory mark |
| 1337 «Элитный» | радиокоды меняют двери | доступ к DATA/метро | электрические сбои и роботы |

Остальные номерные этажи остаются backlog. Не добавлять 20 pockets до того, как эти четыре станут playable.

## FloorInstance вместо enum-раздувания

Система должна описывать pocket floors через определения:

```ts
export interface NumberedFloorDef {
  id: string;
  displayNumber: string;
  title: string;
  entryConditions: string[];
  generatorId: string;
  localRule: string;
  maxDurationMinutes?: number;
  exitRules: string[];
  rewardTags: string[];
  memoryEffect?: string;
}
```

`FloorLevel` остается для больших постоянных этажей. `NumberedFloorDef` создает временный instance с собственным seed, правилами и exit state. Сохранения должны терпеть отсутствие активного instance: если старое сохранение не знает номерной этаж, игрок возвращается к последнему стабильному floor.

## Память и карта

Петля 404 не должна выглядеть как баг. Игра обязана помечать ошибку художественно: пустой номер, неправильный звук лифта, журнал с записью «этаж не найден», NPC, который не помнит поездку. Карта может быть отключена или искажена, но игрок должен иметь альтернативные ориентиры: звук, двери, документы, повторяющиеся комнаты.

Память NPC реализуется как короткие flags/rumors. Не делать сложную систему амнезии. Достаточно: NPC знает/не знает этаж, помнит ложный номер, боится лифта, просит не говорить о 404.

## Самосбор

Самосбор является главным генератором ошибок. Классический вариант повышает шанс 556. Тихий вариант чаще ведет на 404, потому что нет нормального предупреждения. Электрический вариант открывает 1337. Мясной резонанс подменяет 777 слишком уютной мясной безопасностью. Мокрый вариант может отправить в затопленный pocket, если метро-экспаншн уже подключен.

Петля не должна запускаться слишком часто. Это редкое событие, иначе городская легенда станет транспортной системой.

## Контентная структура pocket floor

Каждый pocket состоит из 5-12 комнат, одного локального правила, одного ключевого NPC или следа, одного риска и одного выхода. Это намеренно мало. Номерной этаж должен быть запоминающимся, а не большим.

Пример 404:

| Комната | Функция |
| --- | --- |
| Лифтовой холл без номера | вход, предупреждение |
| Потерянная кладовая | возвращает один потерянный предмет или ложный предмет |
| Пустая очередь | NPC/звуки без тел |
| Дверь с неправильной табличкой | выход, если игрок нарушает карту |
| Архивный тупик | документ для Райсовета/Пустоты |

## Техническая интеграция

Планируемые файлы:

| Файл | Назначение |
| --- | --- |
| `src/data/numbered_floors.ts` | defs для 404/556/777/1337 |
| `src/systems/floor_instances.ts` | entry/exit state, seed, save tolerance |
| `src/systems/elevator_anomalies.ts` | шанс/условия ошибки лифта |
| `src/gen/numbered/floor_404.ts` | generator pocket |
| `src/gen/numbered/floor_556.ts` | generator pocket |
| `src/gen/numbered/floor_777.ts` | generator pocket |
| `src/gen/numbered/floor_1337.ts` | generator pocket |
| `src/data/rumors.ts` | слухи, если rumor system есть; иначе notes/dialogue |
| `src/systems/debug.ts` | открыть конкретную петлю, вывести active instance |

Если система метро уже реализована, она может служить входом. Если нет, лифты и документы работают самостоятельно.

## Производительность и Math LOD

Low: один active pocket за раз, 5-8 комнат, без persistent simulation после выхода.

Middle: четыре pocket defs, entry conditions, local rules, limited memory flags.

High: связи с метро, архивом, слухами, world log, item recovery.

Ultra: visual overkill для каждого номера: 404 без HUD-карты, 556 старые панели П-46, 777 стерильная безопасность, 1337 ламповый киберпанк. Логика все еще finite instance.

## Вертикальный срез

MVP готов, когда игрок может спровоцировать 404, попасть в pocket, понять локальное правило, выйти через неправильную дверь и получить документ/последствие. Остальные номера могут быть stub defs до второго milestone, но первый 404 должен быть полностью playable.

Definition of Done:

| Область | Проверка |
| --- | --- |
| Instance | временный floor создается и безопасно закрывается |
| 404 | есть вход, локальное правило, выход, награда/последствие |
| Лифт | ошибка имеет условия и предвестники |
| Карта | искажение читается как feature, не как баг |
| Самосбор | минимум 2 варианта влияют на шанс/тип петли |

## Риски

Главный риск: номерные этажи станут набором мемов. Контрмера: каждый pocket должен иметь системный крючок и бытовую деталь, не только номер.

Второй риск: игрок решит, что игра сломалась. Контрмера: предвестники, записи, clear exit rules, debug проверки.

Третий риск: save/load сломается на временном этаже. Контрмера: один active instance, seed/state serialization, fallback на стабильный floor.

## Связь с другими экспаншнами

Лифтовая петля использует метро как альтернативный вход, Райсовет как источник документов, Школу как радиослухи, Больницу как память/диагноз, Черный рынок как торговлю потерянными вещами и Пустоту как поздний смысл ошибок.
