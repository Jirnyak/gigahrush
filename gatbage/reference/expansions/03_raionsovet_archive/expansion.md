# Expansion 03: Райсовет и Живой архив

Версия: 0.1 design  
Статус: технический дизайн для разработки  
Роль в линейке: бюрократический systemic expansion  
Базовые источники: `README.md`, `desdoc.md` разделы ADMIN, DATA, документы, фракции, слухи и самосбор

## Назначение

«Райсовет и Живой архив» превращает бюрократию из атмосферы в управляемую, опасную и полезную систему. Игрок получает не только оружие и ПСИ, но и власть через документы: пропуска, справки, печати, акты, журналы, доносы, архивные карточки NPC и приказы на доступ. В гигахруще бумага должна открывать двери не хуже ключа и убивать не хуже монстра.

Экспаншн не добавляет «административный уровень ради кабинетов». В текущей реализации уже есть `MINISTRY`, и документ уважает ограничение: сначала расширяем министерский этаж и связанные комнаты, не требуя нового `FloorLevel.ADMIN`. Новый большой admin-floor допускается позже, если текущий слой перестанет вмещать gameplay.

## Игровая задача

Сейчас многие проблемы игрок решает боем, обходом или торговлей. Этот экспаншн добавляет четвертый путь: легальное или поддельное разрешение. Игрок может получить доступ к складу через справку, снизить штраф через печать, найти NPC через архивную карточку, вытащить человека из очереди через приказ или сломать жизнь персонажу неверным доносом.

Бюрократия должна быть абсурдной, но не бессмысленной. Правила могут быть мерзкими и смешными, однако игрок должен понимать их последствия.

## Основная игровая петля

Игрок приходит в Райсовет за одним простым документом. Его отправляют в очередь, затем за другой формой, затем к печатнику, затем в архив. На каждом шаге есть альтернативы: ждать, подкупить, украсть бланк, подделать подпись, выполнить поручение, применить насилие или найти обходной коридор. Полученный документ меняет реальный доступ: открывает гермодверь, снижает агрессию охраны, дает скидку, разрешает реквизицию или помечает игрока как подозрительного.

Петля строится на выборе цены. Бумажный путь медленнее боя, но открывает долгосрочные последствия и A-Life реакции.

## Мир и места

MVP расширяет `MINISTRY` через контент-модули:

| Место | Функция |
| --- | --- |
| Райсоветская очередь | социальный узел, слухи, NPC с конфликтами |
| Бюро пропусков | легальный доступ к закрытым комнатам |
| Комната печатей | крафт/подделка документов через риск |
| Живой архив | поиск NPC, долгов, старых событий |
| Кабинет взысканий | штрафы, доносы, административные debuffs |
| Узел громкоговорителей | локальные объявления и ложные предвестники |

Расширенная версия добавляет DATA-архив: ламповые ЭВМ, перфокарты, магнитные ленты, цифровые призраки и связь с `VOID`. Это должно прийти после бумажного MVP, иначе киберпанк оторвется от бытового слоя.

## Документ как системный объект

Документ должен быть определением, а не строкой в диалоге:

```ts
export interface DocumentPermitDef {
  id: string;
  title: string;
  issuer: string;
  accessTags: string[];
  expiresHours?: number;
  suspicion: number;
  forgeryDifficulty: number;
  validFloors: FloorLevel[];
  onInspectText: string;
}
```

Для MVP документы могут жить как item/note hybrids. Если текущий inventory не готов к отдельному document type, используется существующий `ItemType` с quest/lore metadata и проверкой по id. Главное: документы должны иметь эффект доступа, цены или риска.

## Живой архив

Архив хранит не всю историю мира, а ограниченные факты: известные NPC, завершенные квесты, последние важные world events, штрафы, пропуска и слухи. Запрос в архиве выполняется через UI/debug/диалог и возвращает короткую карточку. Это не база данных на каждый шаг NPC.

Пример карточки:

| Поле | Смысл |
| --- | --- |
| ФИО/кличка | как NPC числится в системе |
| Последняя зона | агрегированная зона, не точная координата |
| Долги | фракционные или бытовые |
| Опасность | слуховая, может быть ошибочной |
| Доступ | какие документы связаны с NPC |

Архив может ошибаться. Самосбор портит карточки, меняет даты, дублирует людей и создает мертвые записи о живых.

## Фракции и власть

Министерство получает больше власти, но не становится «хорошей» фракцией. Ликвидаторы требуют быстрые приказы и ненавидят очереди. Граждане стоят в очередях и торгуют местом. Культисты подделывают акты на мясные комнаты. Дикие крадут печати. Игрок может стать полезным бюрократическим посредником или врагом системы.

Репутация должна влиять на проверку документов, но без тяжелой социально-правовой симуляции. Достаточно suspicion score и нескольких access tags.

## Самосбор

Самосбор бьет по бюрократии особенно грязно: формы меняют номера, очередь забывает свою цель, громкоговоритель объявляет несуществующий приказ, архив выдает карточку игрока с завтрашней датой смерти. Это не только flavor. Во время или после самосбора документы могут получить флаг `warped`, который делает их сильнее, опаснее или недействительными.

Классический вариант портит печати. Тихий вариант создает ложные приказы без сирены. Электрический ломает DATA-узлы. Мясной резонанс добавляет органические подписи и культовые печати. Мокрый вариант физически размывает бумагу, но может открыть водяные архивные сейфы.

## Техническая интеграция

Планируемые файлы:

| Файл | Назначение |
| --- | --- |
| `src/data/documents.ts` | разрешения, справки, акты, штрафы |
| `src/systems/documents.ts` | проверка access tags, сроков, подделки |
| `src/systems/archive.ts` | bounded lookup по NPC/world facts |
| `src/gen/ministry/raionsovet_queue.ts` | очередь и NPC |
| `src/gen/ministry/live_archive.ts` | архивная зона |
| `src/gen/ministry/stamp_blackroom.ts` | печати и подделка |
| `src/data/notes.ts` | оригинальные тексты форм и приказов |
| `src/systems/debug.ts` | выдать документ, проверить доступ, испортить печать |

Интеграция с `world_log.ts` полезна, но опциональна. Если журнал событий отсутствует или меняется другим агентом, архив работает на локальных quest/NPC facts.

## Производительность и Math LOD

Low: документы проверяются только при интеракции с дверью/NPC; архив возвращает заранее подготовленные карточки; очередь статична.

Middle: suspicion score, сроки действия, несколько активных штрафов, реакция охраны.

High: архив учитывает последние важные события, NPC помнят доносы, документы влияют на цены и доступ к квестам.

Ultra: DATA-архив дает визуальный overkill: лампы, перфоленты, мерцающие таблицы, цифровые призраки. Логика остается lookup-based, без постоянного поиска по миру.

## Вертикальный срез

MVP заканчивается, когда игрок может получить легальный пропуск, подделать альтернативный документ, открыть с их помощью разные доступы и увидеть последствия проверки. В срезе должен быть один случай, где бумага лучше боя, и один случай, где бумага создает проблему.

Definition of Done:

| Область | Проверка |
| --- | --- |
| Документы | минимум 8 document defs с реальным access/risk эффектом |
| Контент | очередь, печатник, архивариус, проверяющий охранник |
| Самосбор | минимум 2 способа порчи или мутации документа |
| Архив | запрос по одному NPC/квестовому факту |
| Баланс | подделка полезна, но повышает suspicion |

## Риски

Главный риск: игрок утонет в бумажной UI-рутине. Контрмера: документы должны быть короткими и проверяться контекстно. Не заставлять игрока вручную сортировать 30 форм ради одной двери.

Второй риск: бюрократия станет шуткой без gameplay. Контрмера: каждый документ имеет механический эффект: доступ, цена, штраф, репутация, квестовый путь или фракционная реакция.

Третий риск: админка сломает Министерство как уже существующий этаж. Контрмера: MVP расширяет `MINISTRY`; новый `ADMIN` рассматривается только как позднее масштабирование.

## Связь с другими экспаншнами

Райсовет выдает санитарные формы для «Грибной смены», разрешения на станционные маршруты для «Метро ошибочной линии», лицензии для «Черного рынка 88», медкарты для «Больничного блока» и акты поставки для «Промзоны концентрата». Это не отдельная бюрократическая шутка, а слой правил над всеми следующими DLC.
