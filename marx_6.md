## Текущая задача
**Туториал (Шаг 2): Скрипт туалета. Обучение механике туалета (bladder) + наказание за неприличие.**

### Контекст задачи
Второй этап стартового туториала обучает игрока справлять нужду.
После того, как игрок на Шаге 1 попил воды из раковины, его параметр `bladder` падает (например, до 15). 
Появляется подсказка/квест: «Нужно в туалет. Найди ванную.»
Дверь в ванную комнату уже открыта (разблокирована по завершении Шага 1).
Внутри ванной находится объект `Feature.TOILET`. 
Игрок должен взаимодействовать с ним (`E` layer action). 
При успешном использовании туалета появляется подсказка: «Чисто и культурно.»

**Естественное обучение (Альтернатива)**: 
В ГИГАХРУЩЕ поощряется естественное обучение через механики мира, а не скрипты. Если игрок решает справить нужду в коридоре (или прямо в комнате), генерируется событие `urination_public`. НПЦ-сосед (marx_11) услышит/увидит это, и его отношение к игроку (relation drop) мгновенно упадет. Это спровоцирует агро (AI aggro), и сосед нападет на игрока. 
Это обучает правилу: «не ссы в коридоре — побьют». 
Этот механизм уже должен быть частично поддержан системно, но ваша задача — убедиться, что событие публикуется, и ИИ реагирует на него в рамках туториальной зоны.

После культурного использования туалета:
Шаг 2 считается выполненным. Следующее действие — разблокировать дверь в столовую (cafeteria door), которая до этого момента находилась в состоянии `DoorState.LOCKED`. Дверь переводится в `DoorState.OPEN` или `DoorState.CLOSED` (но не заперта), позволяя игроку пройти к следующему шагу.

### Конкретные файлы и паттерны
- **`src/systems/tutorial.ts`** (или `src/systems/tutorial_manager.ts`): Реализация шага `TOILET`. Добавление Listener на событие `'use_toilet'` или `'interaction'`.
- **Враждебный НПЦ**: Не нужно скриптовать отдельно — это уже работает через `urination_public` → relation drop → AI aggro. Убедиться что событие публикуется.
- **Стейт дверей**: Управление `DoorState` через `world.doors[cafeteriaDoorId].state = DoorState.CLOSED`.

#### 2. Слой Systems (`src/systems/tutorial.ts` и `src/systems/interactions.ts`)
Обработчик туториала должен слушать систему взаимодействий (`E` action).
```typescript
// Заглушка (Stub) для логики в systems/tutorial.ts
import { publishEvent, subscribeEvent } from '../systems/events';

export function checkTutorialToiletStep(world: World, actionPayload: any) {
  const tutorialState = world.tutorialState;
  
  if (tutorialState.currentStep === TutorialStepId.USE_TOILET) {
    if (actionPayload.type === 'use_toilet' && actionPayload.success) {
      // Игрок успешно использовал туалет
      showHudMessage(TutorialStepConfigs[TutorialStepId.USE_TOILET].successText);
      advanceTutorial(world);
      
      // Разблокировать дверь в столовую
      unlockCafeteriaDoor(world);
    }
  }
}

function unlockCafeteriaDoor(world: World) {
  // Найти дверь в столовую (по ID или тегу, определенному в слое Gen)
  const cafeteriaDoor = world.doors.find(d => d.tags.includes('tutorial_cafeteria_door'));
  if (cafeteriaDoor && cafeteriaDoor.state === DoorState.LOCKED) {
    cafeteriaDoor.state = DoorState.CLOSED; // Или OPEN
    // Обязательно увеличить версию стейта мира для рендера
    world.surfaceVersion++; 
  }
}
```
