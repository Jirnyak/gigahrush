## Текущая задача
**Новые Мобы: Экология Гномов (monster_ecology.ts).**

### Контекст задачи
В задаче `marx_47` был создан сам монстр Гном. В этой задаче необходимо прописать его экологию (привычки, спавн, интеграцию в лор игры).
Экология определяет: на каких этажах он живет, какие комнаты предпочитает (темные склады, вентиляцию), какие слухи о нём ходят среди NPC, и какой вес он имеет при процедурном спавне.

### Конкретные файлы и паттерны
- **`src/data/monster_ecology.ts`**: Добавить запись для `GNOME`.
- **`src/data/rumors.ts`** (или эквивалент): Добавить текстовые лорные слухи о гномах.
- **`src/gen/procedural_floors.ts`** (и генераторы Maintenance / Living): Внедрить `MonsterKind.GNOME` в `monster_mix` с определенным весом (~5%).

### Детальная спецификация по Архитектуре и Реализации (Строго обязательно к исполнению)

#### Шаг 1: Прописка экологии
В `src/data/monster_ecology.ts` добавьте подробное описание.
```ts
// src/data/monster_ecology.ts
import { MonsterKind } from '../core/types';
import { FloorLevel, RoomType } from '../core/constants';

export const ECOLOGY_DATA = {
  // ...
  [MonsterKind.GNOME]: {
    floors: [FloorLevel.MAINTENANCE, FloorLevel.LIVING],
    roomPreference: [RoomType.STORAGE, RoomType.CORRIDOR, RoomType.TECHNICAL],
    lightPreference: 'dark', // Предпочитают неосвещенные участки
    counterplay: 'Гномы быстры, но хрупки. Используйте дробовик на подходе или загоняйте их в узкие коридоры. Они трусливы поодиночке, убейте вожака — остальные разбегутся.',
    lootHint: 'Собирают блестящий мусор: провода, платы, болты. Редко — краденые ювелирные изделия.',
    rumorIds: ['gnome_sighting_1', 'gnome_nest_2']
  },
  // ...
};
```

#### Шаг 2: Добавление слухов (Lore)
Добавьте слухи в словарь, чтобы НПЦ могли говорить о гномах.
```ts
// src/data/rumors.ts
export const RUMORS = {
  // ...
  'gnome_sighting_1': 'Слышал шорох в вентиляции ночью. Опять эти мелкие твари с технического этажа скребутся. Прячь ценности.',
  'gnome_nest_2': 'Говорят, если найти гнездо карликов, там столько хабара натащено... Но они стаей и слона загрызут.',
};
```

#### Шаг 3: Интеграция в пулы спавна
В файлах генерации этажей (например, `src/gen/living/floor.ts` или общих пулах) добавьте гнома.
```ts
// В настройках генератора или файле пулов спавна:
const MAINTENANCE_MONSTER_MIX = [
  { kind: MonsterKind.SPIDER, weight: 30 },
  { kind: MonsterKind.ZOMBIE, weight: 50 },
  { kind: MonsterKind.GNOME, weight: 15 }, // <-- Гномы появляются относительно часто на тех. этажах
  // ...
];

const LIVING_MONSTER_MIX = [
  // ...
  { kind: MonsterKind.GNOME, weight: 5 }, // <-- Редко на жилых этажах (вылазки за лутом)
];
```

#### Шаг 4: Реакция на свет (опционально)
Поскольку экология указывает `lightPreference: 'dark'`, убедитесь, что при процедурном расставлении мобов по комнатам, гномы спавнятся преимущественно в `RoomType.STORAGE` и помещениях с разбитыми/отсутствующими лампами.

### QA и Тестирование
1. Сгенерируйте этаж `MAINTENANCE`.
2. Используя инструмент дебаг-карты, проверьте распределение монстров. Около 15% мобов должны быть гномами, и они должны кучковаться в технических комнатах.
3. Поговорите с NPC в безопасной зоне. Периодически они должны выдавать слух `gnome_sighting_1`.
4. Запустите `npm run typecheck` и `npm run content:audit`.
