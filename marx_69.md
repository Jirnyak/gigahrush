## Текущая задача
**Рендер: Спрайты 128x128 (добавление поддержки в генератор спрайтов).**



### Контекст задачи
Увеличить детализацию ВСЕХ процедурных спрайтов с 64x64 до 128x128 пикселей. Спрайты рендерятся в мире с тем же ВИЗУАЛЬНЫМ размером, но с 4x большей детализацией (как HD-текстуры вместо SD). Это единообразное изменение для ВСЕХ entity — мобов, НПЦ, боссов, предметов. Делать только для боссов — это костыль и неоднородность.

### Конкретные файлы и паттерны
- **`src/render/pixutil.ts`**: Изменить `export const S = 64` → `export const S = 128`. Это ОДНА точка изменения — все генераторы используют `S`.
- **`src/entities/*.ts`**: ВСЕ entity sprite generators автоматически подхватят новый `S` (они уже используют его как размер canvas). Проверить каждый файл — некоторые могут содержать хардкод `64` вместо `S`. Заменить на `S`.
- **`src/render/sprites.ts`**: Спрайт-атлас станет 4x по памяти. Пересчитать количество тайлов в строке атласа. Формула: `atlasWidth = tilesPerRow * S`.
- **`src/render/webgl.ts`**: UV-координаты тайлов в атласе автоматически пересчитаются если они используют `S`. Проверить что нет хардкода `64` или `1/64`.
- **Мобильные (качество)**: Добавить настройку `spriteQuality: 'high' | 'low'`. На мобильных (`navigator.maxTouchPoints > 0`) по умолчанию `'low'` → `S = 64` (как сейчас). На десктопе `'high'` → `S = 128`. Реализовать через `export const S = isMobile ? 64 : 128;` или конфиг.
- **Тестирование**: После изменения обязательно проверить что спрайты рисуются корректно в WebGL (правильные UV, без смещений, артефактов).

#### 1. Модификация Глобальной Константы (Слой `render/` и `data/`)
Определите глобальную константу размера текстуры с учетом настроек или платформы.

```typescript
// В src/render/pixutil.ts
import { GraphicSettings } from '../data/settings'; // Предположим, настройки уже есть

// Определяем базовый размер тайла (S).
// Для десктопов 128 (HD), для тач-устройств по умолчанию 64 (SD).
const isMobile = typeof navigator !== 'undefined' && navigator.maxTouchPoints > 0;
export const DEFAULT_SPRITE_SIZE = isMobile ? 64 : 128;

// Переменная S, используемая во всех генераторах (ранее export const S = 64)
export let S = DEFAULT_SPRITE_SIZE;

export function setSpriteResolution(highRes: boolean) {
    S = highRes ? 128 : 64;
    // После этого нужно пересобрать атлас!
}
```

#### 2. Рефакторинг Хардкода в Сущностях (Слой `entities/`)
Вам необходимо просканировать все файлы в `src/entities/` и `src/render/` на предмет магического числа `64` и заменить его на `S`.


#### 3. Пересчет Атласа (Слой `render/`)
Параметры атласа (ширина, высота, UV координаты) должны динамически вычисляться от `S`.

```typescript
// В src/render/sprites.ts
export function buildSpriteAtlas(generators: Array<() => ImageData>) {
    const tilesPerRow = 16; // Константа количества тайлов в ряд
    const atlasWidth = tilesPerRow * S;
    const rows = Math.ceil(generators.length / tilesPerRow);
    const atlasHeight = rows * S;
    
    const canvas = document.createElement('canvas');
    canvas.width = atlasWidth;
    canvas.height = atlasHeight;
    const ctx = canvas.getContext('2d');
    
    generators.forEach((gen, i) => {
        const x = (i % tilesPerRow) * S;
        const y = Math.floor(i / tilesPerRow) * S;
        ctx.putImageData(gen(), x, y);
    });
    
    // Возвращаем данные для WebGL
    return { canvas, width: atlasWidth, height: atlasHeight, tileSize: S };
}
```
