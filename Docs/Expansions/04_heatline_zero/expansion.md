# Expansion 04: Теплотрасса Ноль

Версия: 0.1 design  
Статус: технический дизайн для разработки  
Роль в линейке: инфраструктурный survival expansion  
Базовые источники: `README.md`, `desdoc.md` разделы HEAT, MAINTENANCE, варианты самосбора, cinematic cheat protocol

## Назначение

«Теплотрасса Ноль» добавляет слой жара, пара, вентилей и давления между жилым гигахрущем и коллекторами. Это expansion про маршруты, риск и контроль среды. Игрок не управляет реалистичной гидравликой; он читает простые состояния: горячо, перекрыто, давление растет, пар закрывает проход, туман временно выжжен, труба сейчас лопнет.

Главный принцип: никакой fluid simulation. Тепло и пар создаются через room flags, таймеры, line-of-sight blockers, урон по зонам и визуальные фейки. Предсказуемость важнее реализма.

## Игровая задача

Текущие коллекторы уже дают трубы, воду и мокрый слой. Теплотрасса добавляет обратный ресурс: жар. Он может убивать, лечить, очищать, открывать маршруты, портить предметы и временно сдерживать туман. Игрок получает новый тип решения: не стрелять в монстра и не искать ключ, а повернуть вентиль, сбросить давление, пройти через остывающий коридор или заманить угрозу в паровой выброс.

Теплотрасса должна быть понятной на уровне одного взгляда: красные лампы, пар, шипение, манометр, ожоги, сухой бетон.

## Основная игровая петля

Игрок попадает в тепловой узел через `MAINTENANCE` или аварийную дверь `LIVING`. На входе часть маршрутов перекрыта паром. Чтобы пройти, нужно найти схему, повернуть вентиль, дождаться короткого окна и решить, стоит ли рисковать. Открытие одного маршрута может закрыть другой, повысить давление в соседней зоне или привлечь существ, которые любят тепло.

После самосбора теплотрасса становится еще ценнее: горячий пар может временно выжечь туман в проходе, но перегрев может взорвать трубы и создать новую опасность. Игрок учится использовать инфраструктуру как оружие, щит и shortcut.

## Мир и места

MVP строится как набор контент-модулей вокруг `MAINTENANCE`:

| Место | Функция |
| --- | --- |
| Главный теплоузел | центральный пульт вентилей и манометр |
| Паровой коридор | timed hazard и первый урок чтения давления |
| Склад асбеста | защитные предметы, бытовой хоррор |
| Аварийная душевая | охлаждение, лечение ожогов, риск зараженной воды |
| Старая котельная | mini-boss или высокий риск |
| Диспетчерская давления | документы, схемы, фракционный контроль |

Поздняя версия может стать отдельным `HEAT` floor instance, но первый срез должен усиливать уже существующие коллекторы.

## Система давления

Давление моделируется дискретно:

```ts
export interface HeatNodeState {
  id: string;
  roomId: number;
  pressure: 0 | 1 | 2 | 3;
  valveState: 'open' | 'closed' | 'jammed' | 'venting';
  heatLevel: 0 | 1 | 2 | 3;
  lastTick: number;
  linkedNodeIds: string[];
}
```

Tick идет редко: 10-30 игровых секунд для активного узла или при взаимодействии. При смене вентиля обновляются только связанные узлы, а не вся карта. Игрок видит результат через двери, пар, температуру, HUD-предупреждение и звук.

## Пар как cinematic cheat

Пар не является объемной средой. Он представлен несколькими дешевыми эффектами:

| Эффект | Реализация |
| --- | --- |
| Закрытие обзора | alpha/tint overlay или wall-column noise в рендере |
| Урон | периодический damage tick в комнате/клетке |
| Шум | spawn/attention event для монстров |
| Очистка тумана | локальное снижение fog значения по room/cell radius |
| Давление | состояние node и манометр в тексте/UI |

Этого достаточно для gameplay. Симуляция температуры каждой клетки запрещена для MVP.

## Монстры и NPC

Новые угрозы не обязаны начинаться с новых entity kinds. Сначала используются модификаторы существующих: горячий ползун, обваренный мертвяк, бетонник с трещинами, тварь, которая боится открытого пара. Позже можно добавить `TubeEel` или «Ламповый», если monster-agent уже создал основу.

NPC: диспетчер тепла, сварщик, санитар ожогового пункта, рабочий с сорванной смены, ликвидатор, который хочет использовать пар как зачистку. Они дают не lore, а решения: схема вентилей, асбест, доступ к душевой, приказ на сброс давления.

## Самосбор

Теплотрасса дает контригру против тумана, но не кнопку победы. Пар может выжечь небольшой участок fog на короткое время. Мокрый самосбор снижает эффективность тепла и повышает риск взрыва. Электрический ломает автоматику вентилей. Мясной резонанс превращает старую котельную в пульсирующий heat-heart, где пар пахнет мясом. Тихий самосбор опасен тем, что игрок принимает шипение труб за обычную аварию.

Самосборный эффект должен быть локальным. Нельзя позволять игроку очистить этаж одним вентилем.

## Техническая интеграция

Планируемые файлы:

| Файл | Назначение |
| --- | --- |
| `src/data/heat_nodes.ts` | типы узлов, пороги, связки |
| `src/systems/heat.ts` | редкие ticks, вентили, урон, fog interaction |
| `src/gen/maintenance/heatline_zero.ts` | первый теплоузел |
| `src/gen/maintenance/boiler_old.ts` | старая котельная |
| `src/data/notes.ts` | схемы вентилей, журналы ожогов |
| `src/render/hud_fx.ts` | если безопасно: короткий heat/fog overlay |
| `src/systems/debug.ts` | давление узла, открыть/закрыть вентиль, сбросить пар |

Интеграция с рендером должна быть минимальной. Если renderer занят другими агентами, визуал MVP делается через существующие текстуры, цвет fog и HUD-сообщения.

## Производительность и Math LOD

Low: 3-5 heat nodes, state changes only on interaction, пар как damage zone без видимого volumetric эффекта.

Middle: редкий pressure tick, linked nodes, разные ожоги, локальная очистка fog.

High: фракции спорят за теплоузлы, NPC используют открытые/закрытые маршруты, документы показывают старые схемы.

Ultra: богатые heat visuals: мерцание, паровые полосы, капли, пульсирующие лампы. Логика не меняется; overkill только в визуале и звуке.

## Вертикальный срез

MVP готов, когда игрок входит в теплотрассу, видит перекрытый паром проход, находит вентиль, меняет состояние, проходит через окно безопасности и может использовать пар против тумана или монстра хотя бы один раз.

Definition of Done:

| Область | Проверка |
| --- | --- |
| Маршруты | минимум 2 вентиля меняют доступность разных проходов |
| Угроза | пар наносит читаемый, ограниченный урон |
| Самосбор | пар временно влияет на fog, но не чистит этаж |
| Контент | диспетчер, схема, ожоговый пункт, старая котельная |
| Debug | команда состояния heat nodes |

## Риски

Главный риск: pressure system перерастет в скрытую инженерную игру. Контрмера: дискретные уровни 0-3 и видимый feedback. Игрок должен понимать, что сделал вентиль, без чтения кода.

Второй риск: пар станет бесплатным оружием. Контрмера: cooldown, перегрев, риск перекрыть собственный маршрут, ограниченный радиус очистки.

Третий риск: визуал будет дорогим. Контрмера: сначала текстуры, tint и HUD; потом, если есть бюджет, procedural steam strips.

## Связь с другими экспаншнами

Теплотрасса питает «Грибную смену» теплом и плесенью, дает аварийные проходы к «Метро ошибочной линии», создает санитарные причины для «Больничного блока» и производственные зависимости для «Промзоны концентрата». Это infrastructure DLC: оно должно сделать старые маршруты новыми.
