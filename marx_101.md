## Текущая задача
**Просадки ФПС (Оптимизация рендера и ИИ)**

### Контекст задачи
Проверить, почему ФПС иногда падает до 40-50, и оптимизировать рендер или ИИ. Профайлинг показывает, что проблема связана с частым пересчетом путей (Pathfinding) и избыточными отрисовками партиклов, когда они находятся вне экрана. Необходимо внедрить Spatial Hash для партиклов и ограничить частоту пересчета путей для НПЦ вне зоны видимости (Distance-based throttling).

### Шаг 2. Внедрение троттлинга для ИИ (AI Throttling)
Для НПЦ, которые находятся далеко от игрока (например, `distance > 30` тайлов), частота обновления пути должна быть снижена до 1 раза в секунду, вместо каждого кадра.
```typescript
// Заглушка для src/systems/ai/pathfinding.ts
export function updateAIPathfinding(world: World, dt: number) {
  for (const npc of world.npcs) {
    const dist = Math.hypot(npc.x - world.player.x, npc.y - world.player.y);
    const updateInterval = dist > 30 ? 60 : 10; // в кадрах
    
    if (world.frameCount % updateInterval === npc.id % updateInterval) {
      recalculatePath(world, npc);
    }
  }
}
```
