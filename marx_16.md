## Текущая задача
**Арена: Система ставок. Экономика (списание денег, расчет победителя, выигрыш).**

### 1. Глубокий контекст задачи и бизнес-логика
Гладиаторские бои на арене (реализованные в marx_15) должны стать экономическим хабом. Игрок может подойти к букмекеру (NPC или Терминал) и сделать ставку на исход битвы. Система должна уметь автоматически оценивать "боевой потенциал" сгенерированных НПЦ, чтобы рассчитывать справедливые коэффициенты. Если игрок угадывает победителя — он получает деньги `(ставка * коэффициент)`. Если ошибается — деньги сгорают. 

Ставки — это транзиентное состояние. Не нужно усложнять систему сохранений. Если игрок сделал ставку и перезагрузил игру посреди боя — ставка сгорает (anti-savescumming механика). Деньги (свойство `money`) сохраняются стандартно.

export function calculateOdds(fighterA: Entity, fighterB: Entity): { oddsA: number, oddsB: number } {
    // Оценка силы: базовое hp, уровень брони, ДПС оружия
    const scoreA = calculateCombatScore(fighterA);
    const scoreB = calculateCombatScore(fighterB);
    
    const total = scoreA + scoreB;
    
    // Фаворит получает меньший коэффициент (ближе к 1.1), аутсайдер - больший (до 3.0+)
    // Формула с маржой букмекера (выплаты < 100% пула)
    const margin = 0.90; // 10% забирает казино
    
    return {
        oddsA: Math.max(1.1, (total / scoreA) * margin),
        oddsB: Math.max(1.1, (total / scoreB) * margin)
    };
}

### 5. Возможные Edge Cases и Подводные Камни
- **Ничья (Оба умерли одновременно от кровотечения/гранаты):** Убедитесь, что логика обрабатывает ничью. По правилам казино, при ничьей ставка обычно возвращается игроку без множителя.
- **Округление:** При расчете выплат используйте `Math.floor()` или `Math.round()`, чтобы в инвентаре не появились дробные рубли (например, `10.5342`).
- **Множественные вызовы:** Защититесь от дабл-кликов в UI, чтобы игрок не смог отправить 10 ставок за долю секунды.
