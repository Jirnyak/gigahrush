# Замечание 2: Автоматическое определение языка

## Проблема
Язык при запуске не определяется через GamePush SDK. Платформа требует, чтобы стартовый язык определялся автоматически методами SDK (с фоллбеком на английский, если нужного языка нет в игре). Ручной экран выбора языка при запуске нужно пропустить.

## План коррекции
1. Изучить интеграцию GamePush SDK (`gp`) при старте игры.
2. Вызывать метод SDK для получения языка пользователя/браузера (согласно документации, например `gp.language()`) до отрисовки главного меню.
3. Передать полученный языковой код в систему локализации (`l10n`).
4. Настроить фоллбек: если полученный язык != `ru`, принудительно ставить `en`.
5. Отключить показ UI экрана с выбором языка при самом первом запуске.

## Что проверить дополнительно
- Прочитать документацию GamePush: как именно называется метод определения языка и какой формат строки он возвращает (`ru`, `ru-RU`, `en` и т.д.).
- Найти место в коде (возможно `main.ts` или `PRCampaign/portal.md`), где сейчас реализовано ожидание пользовательского ввода для выбора локали, чтобы вырезать этот этап безопасно.
