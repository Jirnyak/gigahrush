## Текущая задача
**Туториал: Отключение спавна агрессивных мобов в стартовой квартире.**

### 1. Глубокий контекст задачи и бизнес-логика
Внутри стартовой квартиры, до момента открытия главной двери ключом, игрок находится в "режиме песочницы". В этот период необходимо полностью отключить спавн агрессивных мобов, пространственных аномалий и случайных глобальных событий (таких как faction events, caravans или атаки бандитов). В квартире должны находиться только игрок и несколько сюжетных соседей (например, Семеныч). 

Однако, после прохождения "Самосбора" (Samosbor event), который должен стать мотиватором для побега из зоны, стандартный спавн мелких мобов должен возобновиться, подталкивая игрока покинуть стартовую квартиру.

### 2. Архитектурные требования и изменения по слоям
Проект строго следует пятислойной архитектуре. Нарушение границ слоев недопустимо. Ниже представлены конкретные TypeScript интерфейсы (stubs) для реализации логики.

#### Слой `gen/` (Генераторы)
- **Файл:** `src/gen/design_floors/tutorial_apartments.ts`
При генерации стартовой квартиры генератор должен считывать параметры из `data/` и не вызывать стандартные функции расселения (population placement), которые могут заспавнить монстров в обход лимитов.

```typescript
// TypeScript Stub для src/gen/design_floors/tutorial_apartments.ts
import { DESIGN_FLOOR_POPULATION } from '../../data/design_floor_population';

export function generateTutorialFloor(world: World): void {
    // 1. Генерация базовой геометрии стен и комнат
    generateGeometry(world);
    
    const popConfig = DESIGN_FLOOR_POPULATION['tutorial'];
    
    // 2. Спавн только разрешенных НПЦ
    for (let i = 0; i < popConfig.npcTarget; i++) {
        spawnNeighborNpc(world);
    }
    
    // 3. Форсированная защита от случайного спавна: 
    // Явно обнуляем счетчики, чтобы система не пыталась "доспавнить" недостающих
    world.monstersSpawned = 0;
    world.anomaliesSpawned = 0;
}
```

### 5. Возможные Edge Cases и Подводные Камни
- **A-Life Refiller:** Будьте осторожны с системным восполнением популяции (A-Life refiller). Убедитесь, что он также читает конфигурацию из `DESIGN_FLOOR_POPULATION`.
- **Застревание после Самосбора:** Мобы, заспавненные после Самосбора, не должны быть слишком сильными, иначе игрок не сможет выйти из квартиры. Только Tier 1 мобы (например, мелкая слизь).
