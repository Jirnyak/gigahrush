## Текущая задача
**Туториал: НПЦ-соседи по стартовой квартире.**



### Контекст задачи
В стартовой квартире вместе с игроком находятся 2-3 мирных НПЦ (соседи). Их цель — реагировать на действия игрока (например, если он ведет себя невоспитанно) и обучать социальным нормам.

### Конкретные файлы и паттерны
- **`src/gen/design_floors/tutorial_apartments.ts`** (от marx_10): При спавне НПЦ задать `relation: 1` (минимально дружелюбный).
- **`src/systems/needs.ts`**: urination_public уже публикует event, уже снижает faction relation. Проверить что event доходит до nearby NPC.
- **НПЦ спавн**: 2-3 штуки. Occupation: `'resident'`. Вооружение: нет (кулаки). HP: 80 (можно убить, но больно).
- **Без оружия**: Туториальные НПЦ не должны иметь огнестрельного — только melee. Игрок выживет.

### Гранулярный Технический Roadmap (Для Агентов Jules)

Данный раздел содержит исчерпывающие, пошаговые инструкции для автономных агентов по реализации функционала НПЦ-соседей в стартовой квартире. Каждая подзадача должна быть выполнена строго в указанном порядке, чтобы избежать регрессий и нарушений архитектуры.

#### Этап 1: Определение шаблонов НПЦ (Layer: Data / Entities)
Вам необходимо убедиться, что существуют шаблоны для мирных соседей, которые будут населять стартовую локацию.
1. **Файл:** `src/entities/monster_templates.ts` или `src/entities/npc.ts` (найдите актуальный реестр в `src/entities/` или `src/data/`).
2. **Действие:** Если шаблона соседа нет, создайте его в соответствующем файле данных.
3. **Строгие требования к шаблону:**
   - Идентификатор (`id`): `'tutorial_neighbor'`
   - Имя (`name`): Использовать пул имен или статичное (например, "Сосед").
   - Фракция (`faction`): Привязать к мирной фракции (например, `'residents'` или `'neutral'`).
   - Профессия/Тип (`occupation`): `'resident'`
   - Здоровье (`hp`): Строго `80` (обучение: легко убить, но дает сдачи).
   - Оружие (`weapon`): `null` или базовое melee оружие (кулаки).
   - Инвентарь (`inventory`): Добавить базовый мусорный лут для вознаграждения за килл (пустая бутылка, изолента).

**Пример (TypeScript Stub):**
```typescript
// Ожидаемый вид шаблона в реестре entities/data

  tags: ['tutorial_npc', 'civilian'], // Защита от A-Life миграции
  // ... прочие обязательные поля согласно текущей схеме данных
};
```

#### Этап 2: Интеграция генератора (Layer: Gen)
Необходимо разместить 2-3 НПЦ в процедурно генерируемой (или прекомпилированной) стартовой комнате.
1. **Файл:** `src/gen/design_floors/tutorial_apartments.ts` (или `manifest.ts` в той же папке).
2. **Действие:** Найдите функцию, отвечающую за сборку/спавн содержимого стартовой комнаты.
3. **Логика спавна:**
   - Используйте RNG для определения количества: от 2 до 3 НПЦ.
   - Найдите подходящие walkable тайлы внутри границ (bounding box) стартовой комнаты с помощью `world.cells`. Не спавните их в стенах.
   - Важно: Установите начальное отношение (relation) к игроку. Для туториала оно должно быть нейтрально-дружелюбным (`relation: 1`).
   - Используйте стандартные фабрики спавна (например, `spawnEntity` или `createNpc`). Не мутируйте массивы `world.entities` вручную без обновления индексов.

**Пример (TypeScript Stub):**
```typescript
import { spawnEntity } from '../../systems/spawner'; // Проверьте пути импорта
import { setFactionRelation } from '../../systems/factions';

export function populateTutorialApartment(world: World, roomBounds: BoundingBox) {
  const npcCount = Math.floor(Math.random() * 2) + 2; // Рандом 2..3
  for (let i = 0; i < npcCount; i++) {
    const pos = findFreeWalkableTile(world, roomBounds);
    if (!pos) continue; // Защита от бесконечных циклов
    
    const neighborId = spawnEntity(world, 'tutorial_neighbor', pos);
    
    // Инициализация отношения (чтобы не атаковали сразу)
    setFactionRelation(world, neighborId, 'player', 1);
  }
}
```

#### Этап 3: Система реакций и морали (Layer: Systems)
Соседи должны быть интерактивными и реагировать на антисоциальное поведение (помочиться на пол, драка, кража).
1. **Файл:** `src/systems/needs.ts` или `src/systems/events.ts`.
2. **Действие:** Осуществить перехват событий антисоциального поведения. В `needs.ts` уже может быть событие `urination_public`.
3. **Обработка события:**
   - Подпишитесь на `WorldEvents` (или добавьте обработку в update цикл AI/социальной системы).
   - Если происходит триггер `urination_public`, найдите всех НПЦ в радиусе видимости (используя `world.dist2` и проверку line of sight).
   - Примените пенальти к отношениям (`modifyRelation`).
   - Если отношение падает ниже нуля (вражда), НПЦ должен перейти в состояние атаки (aggro) на игрока.
   - Добавьте визуальный фидбек: публикация события `npc_bark` или `float_text` (например, "Ты что творишь?!").

**Пример (TypeScript Stub):**
```typescript
// Внутри системного цикла обработки событий (systems/social.ts или аналогичный)
export function processSocialEvents(world: World, events: GameEvent[]) {
  for (const event of events) {
    if (event.type === 'urination_public') {
      // Использовать spatial hash или entity_index для оптимизации (Iron Law!)
      const nearbyNpcs = findEntitiesInRadius(world, event.pos, VISION_RADIUS);
      
      for (const entity of nearbyNpcs) {
        if (isTutorialNpc(entity) && hasLineOfSight(world, entity.pos, event.pos)) {
           // Снижение репутации
           modifyRelation(world, entity.id, 'player', -15);
           
           // Вывод реплики в UI
           publishEvent(world, { 
             type: 'float_text', 
             pos: entity.pos, 
             text: 'Фу, извращенец!' 
           });
        }
      }
    }
  }
}
```

#### Этап 4: Боевая тактика безоружных (Layer: Systems / AI)
Если игрок нападает первым или доводит соседей до белого каления.
1. **Файл:** `src/systems/ai.ts` или модули поведения (behavior trees).
2. **Действие:** Реализуйте или проверьте ветку поведения для безоружного (`melee-only`) ИИ.
3. **Логика:**
   - Сосед должен идти на сближение с помощью pathfinding'а, если у него нет дистанционного оружия.
   - Урон от кулаков должен быть минимальным (например, 2-5 HP), чтобы игрок мог выжить и понять механику урона.
   - Убедитесь, что A-Life система не воскрешает туториальных НПЦ. Смерть в туториале должна быть перманентной (persistent fact).

#### Этап 5: Строгое Автоматизированное Тестирование (Layer: Tests)
Все внедренные механизмы должны быть покрыты детерминированными unit-тестами.
1. **Файл:** `tests/systems_tutorial_npcs.test.ts` (создать новый, если нет подходящего агрегатора).
2. **Тестовые сценарии (обязательно реализовать):**
   - **Тест спавна:** Вызвать функцию генерации комнаты. Убедиться, что `world.entities` содержит 2 или 3 новых НПЦ с тегом `tutorial_neighbor`.
   - **Тест социальной реакции:** Заспавнить НПЦ и игрока. Вызвать `publishEvent({ type: 'urination_public' })`. Проверить, что `getRelation()` НПЦ к игроку стал ниже начального значения.
   - **Тест агрессии:** Довести relation до отрицательного значения вручную. Проверить, что при следующем тике AI, НПЦ переходит в состояние `STATE_COMBAT` с целью `player`.
