## Текущая задача
**Улучшение процедурных этажей**

### Контекст задачи
Допилить сшивание модулей и коридоры в генераторе процедурных этажей. Улучшить алгоритм стыковки, чтобы процедурные этажи (нечетные `Z`) выглядели как единый органичный лабиринт, а не набор неряшливо брошенных прямоугольников. Нужно проработать соединения (corridors), чтобы они гарантированно соединяли сгенерированные комнаты без тупиков и непроходимых зон, сохраняя гермодвери и лифты.

### Шаг 2. Разработка улучшенного алгоритма стыковки
1. Вместо случайных пропилов, используйте граф минимального остовного дерева (MST) для соединения центров всех размещенных комнат.
2. Для каждого ребра MST постройте коридор с помощью L-образных переходов (Manhattan) или A*, учитывающего вес клеток (предпочтение уже раскопанным клеткам, чтобы коридоры сливались).
3. Убедитесь, что коридоры учитывают зацикленность тора (`world.wrap`, `world.dist2`).

```typescript
// Stub: src/gen/corridors.ts
```

### Шаг 4. Проверка связности (Flood Fill)
1. В конце генерации процедурного этажа обязательно запустите Flood Fill из клетки с лифтом.
2. Если какие-то комнаты недостижимы, генератор должен либо создать дополнительный коридор к ближайшей достижимой точке, либо стереть недостижимую комнату.
3. Это критически важно для Iron Law: избегать рантайм паники ИИ из-за недостижимости целей.
