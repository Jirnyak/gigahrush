## Текущая задача
**Туториал: Стартовая запертая квартира (начало любой Новой Игры).**



### Контекст задачи
Туториал теперь ВШИТ в начало ЛЮБОЙ игры. Никаких отдельных режимов. При старте «Новой игры» игрок спавнится в изолированной (запертой) стартовой квартире.

**Карта**: Несколько комнат (спальня, туалет/ванная, кухня, коридор). Входная дверь заперта (LOCKED), ключ спрятан или должен быть получен. Внутри есть раковина, унитаз, кровать. Система расширяема.

**Сценарий туториала**: Игрок удовлетворяет первичные потребности (еда, вода, туалет), взаимодействует с окружением (возможно, с соседями), находит ключ и открывает главную дверь. После этого начинается обычная игра с ОД (обычным движением/динамикой).

### Конкретные файлы и паттерны
- **`src/gen/design_floors/tutorial_apartments.ts`** [НОВЫЙ]: Маленький этаж. 5-6 комнат + коридор + гермодверь. Все внешние двери LOCKED без keyId (неоткрываемые). Используйте `communal_ring.ts` как пример масштаба.
- **`src/data/design_floors.ts`**: `{ id: 'tutorial', hudName: 'Туториал', baseFloor: FloorLevel.LIVING }`. НЕ добавлять в route deck — это отдельный режим.
- **`src/gen/design_floors/manifest.ts`**: Регистрация генератора.
- **Спавн игрока**: Начальные stats: `hunger: 30, thirst: 20, bladder: 15` (низкие, чтобы нужды были срочными).

**Этап 1: Интеграция Данных и Регистрация Уровня (Слой `data/`)**
- В файле `src/data/design_floors.ts` (или аналогичном) необходимо добавить определение туториального этажа. 
- Обратите внимание, что туториал не должен попадать в обычный список маршрутов (`routeDeck`), так как вызывается только при старте новой игры как одноразовая инстанция.
- **Stub (Пример регистрации):**
```typescript
// В src/data/design_floors.ts
export const TUTORIAL_FLOOR_ID = 'tutorial';

export const designFloors: Record<string, DesignFloorDef> = {
  // ... существующие этажи
  [TUTORIAL_FLOOR_ID]: {
    id: TUTORIAL_FLOOR_ID,
    hudName: 'Запертая Квартира',

```
- В `src/gen/design_floors/manifest.ts` импортировать генератор туториала и привязать его к ключу `tutorial`.

**Этап 2: Строгая Генерация Геометрии Квартиры (Слой `gen/`)**
- Создать новый модуль `src/gen/design_floors/tutorial_apartments.ts`.
- Реализовать основную функцию `generateTutorialApartment(world: World, seed: number)`.
- **Геометрия:** Сгенерировать небольшую закрытую локацию размером примерно 30x30 клеток. Построить абсолютно непроницаемый внешний гермо-периметр (использовать `WallType.HERMETIC` или эквивалентный бетон), чтобы игрок гарантированно не мог покинуть зону до завершения скрипта.
- **Планировка (Хардкод разрешен для структуры):** В отличие от процедурных этажей, здесь разрешено использовать статический 2D-шаблон (массив строк) для разметки стен, пола и дверей, чтобы обеспечить идеальную читаемость пространства и контроль над игроком.
- **Двери и Блокировки:** 
  - Разместить внутренние двери для комнат.
  - Входная (выходная) гермодверь должна быть заперта (`locked: true`). Ей необходимо присвоить уникальный `keyId: 'tutorial_main_key'`.

**Этап 3: Размещение Интерактивных Объектов и Предметов (Слой `gen/` и `data/`)**
- Спальня: кровать (`BED` из `data/interactives.ts`), терминал (для базового лора и знакомства с сетью Демос).
- Кухня: раковина (`SINK`), холодильник (`FRIDGE`).
- Ванная: унитаз (`TOILET` — с учетом логики гигиены, реализованной в задаче marx_2).
- Размещение начального лута: банка тушенки (для голода), бутылка воды (для жажды).
- **Сюжетный ключ:** Разместить `tutorial_main_key` на полу, либо в инвентаре определенного контейнера (например, куртки в прихожей). Предмет `tutorial_main_key` должен быть предварительно добавлен в `src/data/items.ts` с типом `ItemType.KEY`.

**Этап 4: Система Туториальных Квестов (Слой `systems/` и `data/`)**
- Внедрить обучение через систему квестов (Side Quests). В `src/data/plot.ts` зарегистрировать новый квест `quest_tutorial_escape`.
- Этапы квеста:
  1. Осмотреться (базовая задержка, стартовое сообщение).
  2. Утолить голод и жажду (слушать эвент `EVENT_ITEM_CONSUMED` или чекать изменение статов в `systems/quests.ts`).
  3. Справить нужду (эвент использования унитаза `EVENT_INTERACT_TOILET`).
  4. Найти выход (попытка открыть `locked` дверь вызывает подсказку: "Дверь заклинило. Нужен ключ").
  5. Найти ключ (поднятие предмета `tutorial_main_key` вызывает эвент взятия).
  6. Покинуть квартиру (открытие входной двери завершает квест).
- **Индикация:** Каждый этап должен сопровождаться четкими нотификациями на экране, использующими стандартную UI шину. Обязательно добавьте пояснительные тексты для каждого шага.

**Этап 5: Инициализация Игрока и Привязка к Геймплейному Циклу (Слой `systems/` / `main.ts`)**
- Модифицировать логику начала **Новой Игры** (вызов `worldInit` или эквивалентный). При старте принудительно загружать этаж `tutorial` вместо первого этажа стандартного маршрута.
- Сразу после загрузки уровня, принудительно задать стартовые статы игроку, чтобы заставить его действовать:
```typescript
// Стартовый стресс-тест для игрока в main.ts или инициализаторе:
world.player.stats.hunger = 30;  // Критический уровень
world.player.stats.thirst = 20;  // Критический уровень
world.player.stats.bladder = 15; // Требует срочной разрядки
```
- Снаружи квартиры (сразу за главной дверью) расположить переходную зону — например, работающий лифт или спуск на лестничную клетку, взаимодействие с которыми вызывает стандартный переход на `FloorLevel.LIVING` (настоящий 1-й этаж игры).

**Этап 6: Ограничения, Валидация и Изоляция**
- **Изоляция A-Life:** Строго запретить спавн бродячих NPC (соседей, торговцев, сектантов) внутри туториала. Используйте маски зон `aptMask` или добавьте проверку `isTutorial` в генератор популяции `src/gen/population_placement.ts`.
- **Изоляция Монстров:** Никаких агрессивных мобов в туториале быть не должно. Это чистая безопасная зона для освоения управления. Никаких случайных генераций аномалий.
- **Локализация:** Все тексты туториальных квестов, названий ключей и нотификаций должны быть корректно добавлены в словари и поддерживать извлечение через `npm run l10n:extract`. Никакого жесткого захардкоженного русского текста прямо в рендере.
