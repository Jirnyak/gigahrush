    id: TutorialStep;

    successLog: string;
    targetEvent: string; // На какое событие реагировать (напр. 'PLAYER_DRINK')

                 door.state = DoorState.OPEN; // Разблокировка и открытие
                 publishEvent('DOOR_UNLOCKED', { doorId: door.id });
             }
        }
    },
    // ... заглушки для остальных шагов (будут реализованы в других задачах) ...

#### 3. Требования к Системе Сохранения (Save/Load)
- **Схема сохранения (`save.md`)**: Состояние туториала (`tutorial.active` и `tutorial.currentStep`) должно сериализоваться в секцию данных игрока (в `localStorage`).
- При загрузке игры (load), если туториал `active`, он должен восстановить подписки на события для текущего `currentStep`.
- **Санитизация**: Проверять `currentStep` на валидные значения Enum `TutorialStep`, чтобы избежать крашей при загрузке старых сейвов.
