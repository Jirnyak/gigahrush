## Текущая задача
**Туториал: Выход в основную игру (начало ОД).**

### 1. Глубокий контекст задачи и бизнес-логика
Как только игрок открывает главную дверь квартиры ключом или переживает Самосбор, он должен выйти из "режима песочницы" и перейти в основную игру. Этот переход сопровождается отключением флага `tutorialMode`, переключением этажа на стандартный стартовый (обычно `LIVING` z=0), и включением автосохранения. Важно: статистика игрока (HP, собранный инвентарь) из туториала ДОЛЖНА сохраниться и перенестись в основную игру. С этого момента начинается "Основная Динамика" (ОД) со спавном мобов, случайными событиями и пермадетом с записью в сейв.

### 2. Архитектурные требования и изменения по слоям

#### Слой `systems/` (Системы и Логика)
- **Файл:** `src/systems/tutorial.ts`
Вам необходимо реализовать функцию завершения туториала. Эта функция должна аккуратно снимать ограничения песочницы и инициировать переход в нормальный игровой цикл.

```typescript
// TypeScript Stub для src/systems/tutorial.ts
import { switchFloor } from './floor_transition';
import { preservePlayerStats, forceAutoSave } from './save_runtime';

export function completeTutorial(world: World): void {
    // 1. Отключаем флаг песочницы
    world.tutorialMode = false;
    
    // 2. Получаем текущие данные игрока, чтобы перенести их
    const playerEntity = world.player;
    
    // 3. Переход на стартовый этаж (Жилая зона, z=0)
    // switchFloor должна сама очистить старый этаж, но оставить playerEntity
    switchFloor(world, 'LIVING_Z0');
    
    // 4. Форсируем первое сохранение (теперь, когда tutorialMode = false, оно сработает)
    forceAutoSave(world);
    
    // 5. Публикуем событие для UI и ачивок
    publishEvent('TUTORIAL_COMPLETED', { timestamp: Date.now() });
}
```

- **Интеграция с Самосбором (`src/systems/samosbor.ts`):**
Туториал может закончиться двумя путями: игрок сам вышел в дверь, либо пришел Самосбор, и игрок переходит на другой этаж в рамках aftermath-фазы. Обе ветки должны обрабатываться.

```typescript
// TypeScript Stub для src/systems/samosbor.ts
export function onSamosborEndTransition(world: World): void {
    if (world.tutorialMode) {
        // Если самосбор закончился в туториале, форсируем выход в основную игру
        completeTutorial(world);
    } else {
        // Стандартная логика смены этажа после Самосбора
        processStandardSamosborTransition(world);
    }
}
```

#### Слой `core/` (Состояние)
- **Управление игроком:** Убедитесь, что механизм `switchFloor` не пересоздает объект `Player` с нуля, а сохраняет его текущее HP, голод и `inventory`. Если он пересоздает, напишите метод `transferPlayerData(oldPlayer, newPlayer)`.

### 3. Пошаговый план внедрения (Action Plan)
1. **Реализация `completeTutorial`:** Создайте функцию в `tutorial.ts`, которая отключает флаг и вызывает смену этажа.
2. **Перенос состояния:** Проверьте систему смены этажей (`floor_transition.ts`). Убедитесь, что инвентарь и HP сохраняются при переходе с туториального этажа.
3. **Хук в Самосбор:** Внедрите вызов `completeTutorial` в фазу завершения Самосбора (`samosbor.ts`).
4. **Хук на Дверь:** Если игрок открывает главную дверь квартиры ключом (обработчик взаимодействия в `interactions.ts`), также вызывайте `completeTutorial`.
5. **Включение Сейвов:** Убедитесь, что вызов `forceAutoSave()` сразу создает валидный `localStorage` файл, чтобы если игрок закроет вкладку через секунду после выхода, игра была сохранена.

### 4. План тестирования и Валидации (Validation Rules)

### 5. Возможные Edge Cases и Подводные Камни
- **Двойной вызов перехода:** Игрок может открыть дверь в тот момент, когда Самосбор уже переводит его. Защитите функцию `completeTutorial` от двойного вызова (например, проверкой `if (!world.tutorialMode) return;`).
- **Смерть во время перехода:** Убедитесь, что во время `switchFloor` игрок неуязвим на долю секунды, чтобы шальной урон не убил его до того, как сработает сейв.
