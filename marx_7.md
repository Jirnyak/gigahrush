## Текущая задача
**Туториал: Скрипт столовой. Обучение еде и подбору предметов.**

### Контекст задачи
События происходят на специальном туториальном этаже (Tutorial Floor).
Предыдущий шаг: Игрок посетил туалет. В результате `hunger` (голод) поднимается до 30.
Появляется подсказка: «Голод. Сходи в столовую.»
В столовой должен быть стол (`Feature.TABLE`), на котором лежит еда (item `'bread'` или `'canned_food'`).
Игрок должен подойти к столу, подобрать предмет (клавиша E) и использовать его из инвентаря.
После этого появляется подсказка: «Сытно! Теперь на работу.»
В качестве награды (или следующего шага) — разблокируется дверь на завод.

**Чему это учит:**
- Подбор предметов (нажатие E возле лута).
- Работа с инвентарем (понимание, что предметы сохраняются).
- Использование consumable (еды).

## Детальная спецификация и RoadMap (Минимум 100 строк подробного плана)

### Фаза 1: Анализ и Подготовка
1. Изучите `src/data/items.ts` на предмет наличия предметов `'bread'` и `'canned_food'`. Если их нет — добавьте их как валидные расходники (consumables) с эффектом снижения голода.
2. Проанализируйте `src/systems/tutorial.ts` (или аналогичный файл стейт-машины туториала).
3. Проверьте генерацию туториального этажа `src/gen/tutorial_floor.ts` (или аналогичного) для понимания, где находится столовая и как спавнить там предметы на столе.

### Фаза 2: Дополнение Data слоя (`src/data/`)
1. **Предметы**: Убедитесь, что `ItemDef` для еды настроен правильно. Он должен генерировать событие или вызывать эффект при использовании (consume).
2. **Состояния туториала**: Добавьте новые константы/перечисления для шагов туториала (если используется Enum или строковые константы).
   ```typescript

       // ... предыдущие шаги ...
       GO_TO_CANTEEN = 'GO_TO_CANTEEN',
       PICKUP_FOOD = 'PICKUP_FOOD',
       EAT_FOOD = 'EAT_FOOD',
       GO_TO_FACTORY = 'GO_TO_FACTORY'
   }
   ```
3. **Подсказки (Scenarist/Text)**: Добавьте строки для подсказок в локализацию или константы туториала.
   - `TUTORIAL_HINT_CANTEEN`: "Голод. Сходи в столовую."
   - `TUTORIAL_HINT_FACTORY`: "Сытно! Теперь на работу."

### Фаза 3: Генерация мира (`src/gen/`)
1. **Размещение лута**: В скрипте генерации столовой (`spawnCanteen` или подобном) разместите `Feature.TABLE`.
2. Поверх стола (на тех же координатах x, y) разместите Entity предмета еды.
3. Убедитесь, что предмет подбираем. Если для предметов используется отдельная логика, используйте её:
   ```typescript

       world.items.push({
           type: 'bread',
           x, 
           y,
           pickupable: true
       });
       // Добавьте логику связывания с столом, если это требуется движком
   }
   ```
4. **Дверь завода**: Дверь, ведущая в заводской отсек, должна быть сгенерирована с флагом `locked: true` и иметь уникальный тег/ID (например, `door_factory`).

### Фаза 4: Системы и Логика (`src/systems/`)
1. **Слушатель инвентаря и использования предметов**:
   Туториал должен реагировать на системные события. Не пишите хардкод проверки каждый кадр.
   Используйте шину событий (Events) или хуки на действия.
   ```typescript
   // Пример заглушки для src/systems/tutorial.ts
   import { subscribeEvent } from '../core/events';

   export function initCanteenTutorial() {
       subscribeEvent('item_picked_up', (evt) => {
           if (currentTutorialStep === TutorialStep.PICKUP_FOOD && 
              (evt.itemId === 'bread' || evt.itemId === 'canned_food')) {
               advanceTutorial(TutorialStep.EAT_FOOD);
               showHint('Открой инвентарь и съешь это.');
           }
       });

       subscribeEvent('item_consumed', (evt) => {
           if (currentTutorialStep === TutorialStep.EAT_FOOD) {
               advanceTutorial(TutorialStep.GO_TO_FACTORY);
               showHint('Сытно! Теперь на работу.');
               unlockFactoryDoor();
           }
       });
   }
   ```
2. **Снижение голода (если требуется принудительно)**:
   При переходе в стейт `GO_TO_CANTEEN` установите показатель голода игрока на 30 (или добавьте 30 единиц голода), чтобы спровоцировать механику, если это не делается на предыдущем шаге туалета.
3. **Разблокировка двери**:
   Функция `unlockFactoryDoor()` должна находить дверь по тегу/ID `door_factory` и менять ей `locked: false`, после чего вызывать событие обновления навигационной сетки или графики (например, `bumpSurfaceVersion`).
