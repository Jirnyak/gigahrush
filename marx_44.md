## Текущая задача
**Баланс: Переработка лут-таблиц (контекстный дроп, адекватные шансы).**

### Контекст задачи
Лут в игре должен стать контекстным, чтобы создавать у игрока логичные ожидания (например, медикаменты ищутся в медпунктах).
Принципы новой системы:
1. **Контекст комнаты:** `RoomType.KITCHEN` → еда; `RoomType.STORAGE` → ресурсы; `RoomType.MEDICAL` → химия/PSI.
2. **Контекст этажа:** Living → бытовуха, Ministry → документы/ключи, Hell → мощное оружие/патроны.
3. **Diminishing returns (дефицит):** Если комната отмечена как «обысканная» (`looted = true`), шанс найти ценный предмет падает на 80%.
4. **Scarcity-driven:** Оружие и патроны крайне редки на стартовых этажах, но обильны на поздних.
5. Никогда не дропать plot-critical (сюжетные) предметы через систему случайного лута.

### Конкретные файлы и паттерны
- **`src/systems/procedural_loot.ts`**: Основная система генерации лута. Требует глобального рефакторинга весов и фильтров.
- **`src/data/items.ts`**: Использовать `spawnRooms` и `tags` для фильтрации контекста.
- **`src/core/world.ts` / `src/core/room.ts`**: Проверить наличие флага `isLooted` у комнаты или добавить его.

#### Шаг 2: Настройка весов (Weights Tuning)
Предметы в `data/items.ts` должны иметь тэги (`'food'`, `'med'`, `'ammo'`). В лут-генераторе создайте динамическую модификацию весов.
```ts
// Пример динамической модификации веса предмета
function getItemWeight(itemDef: ItemDef, context: LootContext): number {
  let weight = itemDef.baseSpawnWeight || 10;

  // Усиливаем профильные предметы
  if (context.roomType === RoomType.MEDICAL && itemDef.tags.includes('med')) weight *= 5;
  if (context.roomType === RoomType.KITCHEN && itemDef.tags.includes('food')) weight *= 5;
  if (context.roomType === RoomType.STORAGE && itemDef.tags.includes('resource')) weight *= 3;

  // Влияние этажа на патроны и оружие
  if (itemDef.tags.includes('weapon') || itemDef.tags.includes('ammo')) {
    if (context.floorLevel === FloorLevel.LIVING) weight *= 0.1; // Почти не бывает
    if (context.floorLevel === FloorLevel.HELL) weight *= 3.0; // Обильно
  }

  // Исключаем плот-критикал предметы
  if (itemDef.tags.includes('plot_critical')) return 0;

  return weight;
}
```

#### Шаг 3: Обновление генераторов этажей
При спавне контейнеров или мусора на полу, генераторы (в папке `src/gen/`) должны вызывать `generateProceduralLoot`, передавая правильный `context`.
```ts

  floorLevel: currentFloorLevel,
  roomType: room.type,
  danger: getFloorDanger(currentFloorLevel),
  isLooted: room.flags.includes('looted')

const itemsToSpawn = generateProceduralLoot(ctx);
```
