### 2. Модификация WebGL Шейдеров
Откройте основной фрагментный шейдер в `src/render/webgl.ts` (или `.glsl` файл).

- **Цветокоррекция:** Плавно сдвигайте цветовую палитру в красный/пурпурный спектр пропорционально `u_SamosborIntensity`.
- **Туман (Fog):** Цвет тумана (затемнение вдаль) должен менять цвет с черного на багровый.
- **Глитч:** Добавьте небольшое смещение UV координат на основе функции шума `fract(sin(u_Time))`, которое включается короткими вспышками.

```glsl
// GLSL Snippet
uniform float u_SamosborIntensity;
uniform float u_Time;

void main() {
    // ... basic rendering ...
    vec3 baseColor = color.rgb;
    
    // Сдвиг в красное
    vec3 samosborColor = mix(baseColor, vec3(baseColor.r + 0.3, baseColor.g * 0.5, baseColor.b * 0.5), u_SamosborIntensity);
```

### 3. Синхронизация с системами
- В `main.ts` или основном игровом цикле, прочитайте интенсивность Самосбора из `world.samosbor` или событий.
- Передайте это значение в `updateSamosborUniforms` перед `gl.drawArrays`.

### 4. Моргание света
- Самосбор часто сопровождается перебоями электричества. В `src/render/light.md` (или соответствующем коде света) добавьте умножение радиуса/яркости источников света на мерцающую функцию (sine + noise), когда `intensity > 0.3`.

### 5. Производительность (Iron Law)
- Глитчи и цветокоррекция должны быть скомпилированы в один проход (single pass shader). Никаких дополнительных буферов кадров (FBO) для пост-процессинга, если можно обойтись без них.
- Если `intensity == 0.0`, используйте `if` или математику, чтобы шейдер не делал лишнюю работу. Для мобилок (GLES) `mix` работает быстро.
