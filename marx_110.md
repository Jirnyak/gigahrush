## Текущая задача
**Автоматическая экипировка и Quick-slots**

### Контекст задачи
Игроки жалуются на то, что после поднятия нового расходника или оружия нужно обязательно открывать инвентарь для назначения на быстрый слот (Quick Slot). Требуется внедрить систему: если слот пуст, подходящий предмет автоматически назначается при поднятии. Плюс, добавить визуальные горячие клавиши.

### Конкретные файлы и паттерны
- **`src/systems/inventory.ts`**
- **`src/systems/items.ts`**
- **`src/render/hud.ts`**
- **`src/input.ts`**

## Детальный план реализации (Шаги для Агента Jules)

### Шаг 1. Расширение логики инвентаря
В `src/systems/inventory.ts` обновите функцию `pickupItem(world, entityId, item)`.
После добавления предмета в инвентарь проверяйте пустые слоты (Quick Slots).
```typescript
// Заглушка
export function autoAssignQuickSlot(world: World, entityId: number, item: Item) {
  const inv = world.entities[entityId].inventory;
  if (item.type === 'CONSUMABLE' || item.type === 'WEAPON') {
     const emptySlot = inv.quickSlots.findIndex(slot => slot === null);
     if (emptySlot !== -1) inv.quickSlots[emptySlot] = item.id;
  }
}
```

### Шаг 2. Опциональность системы
Некоторые игроки могут ненавидеть автоэкипировку. Добавьте флаг `autoEquipEnabled: boolean` в настройки игры (см. `src/systems/settings.ts` или аналогичный модуль сохранения конфигурации).

### Шаг 3. Приоритет замены
Если у игрока закончились бинты (Quick Slot 1 опустел), а он подбирает аптечку, игра должна умно поставить её на освободившийся слот. Реализуйте "Память слотов" или группировку по типу в `inventory.ts`.

### Шаг 4. Обновление интерфейса (HUD)
Отрисовывайте на экране (внизу) панель с горячими клавишами `[1] [2] [3] [4]`.
В `src/render/hud.ts` добавьте небольшие иконки и количество оставшихся зарядов (stack size) для каждого слота.

### Шаг 5. Связка с вводом (Input)
Обновите `src/input.ts`, чтобы нажатие клавиш 1-4 вызывало `useItemFromQuickSlot(world, PLAYER_ID, slotIndex)`.
Убедитесь, что это действие требует минимального времени хода (Action Points), если они применяются.

### Шаг 6. Валидация
Протестируйте ситуацию: подбор 10 одинаковых аптечек подряд.
Они должны стакаться в инвентаре, и счетчик на слоте быстрого доступа должен увеличиваться, а не занимать все 4 слота.
Напишите unit-тест `tests/systems/inventory.test.ts` для проверки этого краевого случая.

### Шаг 7. Финальный прогон
Выполните `npm run typecheck`, `npm run content:audit`. Проверьте работу хоткеев на десктопе и тапом на мобильной версии. Оформите PR.
